الإيماءات في Hyprland — الفكرة الأساسية

Hyprland يدعم إيماءات التراك باد بشكل مدمج. بدون أي برنامج إضافي، يمكنك تعيين حركات الأصابع على التراك باد لتنفّذ أي أمر تريده — من تبديل الـ workspaces إلى إغلاق النوافذ والتكبير والتصغير.

البنية الأساسية للإيماءة:

gesture = عدد_الأصابع, الاتجاه, الأمر, الخيارات
  • عدد الأصابع: 3 أو 4 (الأصابع 1 و2 محجوزة للتمرير وتحريك المؤشر)
  • الاتجاه: l يسار، r يمين، u أعلى، d أسفل، horizontal للأفقي، vertical للعمودي
  • الأمر: workspace، close، fullscreen، dispatcher، وغيرها
⚠️ إصبعان فقط (2) لا يمكن استخدامهما للإيماءات المخصصة — محجوزان للتمرير الطبيعي. استخدم 3 أو 4 أصابع.

ملف الإيماءات

يمكنك وضع الإيماءات في ~/.config/hypr/hyprland.conf مباشرةً أو في ملف منفصل تستورده مثل ~/.config/hypr/gestures.conf: 

# في hyprland.conf
source = ~/.config/hypr/gestures.conf

الإيماء الأول — تبديل الـ Workspaces

الأكثر فائدة لمستخدمي اللابتوب — مسح أفقي بثلاثة أصابع:

# gestures.conf

# مسح أفقي بـ 3 أصابع = تبديل workspaces
gesture = 3, horizontal, workspace

لاحظ أن horizontal يغطي الاتجاهين معاً — اليسار واليمين — بشكل تلقائي. لا تحتاج سطرين.

الإيماء الثاني — إغلاق النافذة

مسح للأسفل بثلاثة أصابع مع مفتاح modifier لإغلاق النافذة النشطة:

# مسح للأسفل بـ 3 أصابع + Alt = إغلاق النافذة
gesture = 3, d, close, alt

المفتاح alt هنا مهم — يمنع الإغلاق العرضي. تحتاج تضغط Alt أثناء المسح. يمكن تغييره لـ super أو ctrl حسب تفضيلك.

الإيماء الثالث — التكبير للشاشة الكاملة

# مسح للأعلى بـ 3 أصابع + Super = fullscreen
gesture = 3, u, fullscreen, super

# للخروج من fullscreen نفس الإيماء

يمكن إضافة scale لتسريع أنيميشن الـ fullscreen:

# scale:1.5 تعني الأنيميشن أسرع بـ 1.5x
gesture = 3, u, fullscreen, super, scale:1.5

الإيماء الرابع — تشغيل تطبيق بأربعة أصابع

أربعة أصابع للأسفل تفتح الطرفية (أو أي تطبيق تختاره):

# 4 أصابع للأسفل = فتح kitty
gesture = 4, d, dispatcher, exec, kitty

# أو فتح المشغّل
gesture = 4, u, dispatcher, exec, fuzzel

لاحظ الفرق: حين تستخدم dispatcher كأمر، تضيف بعده اسم الـ dispatcher ثم الـ argument — وهنا المهم إضافة فاصلة بعد exec: 

# صح ✅
gesture = 4, d, dispatcher, exec, kitty

# خطأ ❌ — لن يعمل
gesture = 4, d, dispatcher, exec kitty

قائمة الـ Dispatchers المتاحة للإيماءات

يمكن استخدام أي dispatcher من قائمة Hyprland مع الإيماءات:

  • workspace — التنقل بين الـ workspaces
  • close — إغلاق النافذة النشطة
  • fullscreen — تكبير/تصغير الشاشة الكاملة
  • togglefloating — تبديل وضع التعويم
  • movewindow — تحريك النافذة
  • resizeactive — تغيير حجم النافذة
  • togglespecialworkspace — فتح/إغلاق الـ special workspace
  • exec — تنفيذ أمر أو برنامج
# أمثلة متنوعة
gesture = 3, l, dispatcher, workspace, e-1
gesture = 3, r, dispatcher, workspace, e+1
gesture = 4, u, dispatcher, togglespecialworkspace
gesture = 4, l, dispatcher, movewindow, l
gesture = 4, r, dispatcher, movewindow, r

الملف الكامل الجاهز

# ~/.config/hypr/gestures.conf
# ===================================
# إيماءات التراك باد
# ===================================

# تبديل الـ workspaces — 3 أصابع أفقي
gesture = 3, horizontal, workspace

# إغلاق النافذة — 3 أصابع للأسفل + Alt
gesture = 3, d, close, alt

# Fullscreen — 3 أصابع للأعلى + Super
gesture = 3, u, fullscreen, super, scale:1.2

# فتح المشغّل — 4 أصابع للأعلى
gesture = 4, u, dispatcher, exec, fuzzel

# Special Workspace — 4 أصابع للأسفل
gesture = 4, d, dispatcher, togglespecialworkspace

# تعويم النافذة — 4 أصابع للأسفل + Alt
gesture = 4, d, dispatcher, togglefloating, alt

ملاحظات مهمة للاستخدام

  • الإيماءات ليست بنفس نعومة macOS — هناك تأخير طفيف في بداية التعرف على الإيماءة
  • تأكد أن لا إصبع إضافي يلمس التراك باد عند تنفيذ الإيماءة
  • إذا لاحظت أن الأنيميشن لا يعمل بشكل صحيح أثناء الـ gesture، هذا طبيعي — الـ workspace يتبدل لكن الأنيميشن قد يكون غير مكتمل
  • يمكن ضبط حساسية التراك باد عبر input { touchpad { } } في إعدادات Hyprland
# في hyprland.conf — إعدادات التراك باد
input {
  touchpad {
    natural_scroll = true
    scroll_factor = 0.8
    tap-to-click = true
    disable_while_typing = true
  }
}

مشكلة إيماءات سماعات البلوتوث

إذا كانت عندك سماعات بلوتوث لاسلكية — AirPods أو Sony WF أو Samsung Galaxy Buds — ستلاحظ أن الـ tap للتشغيل/الإيقاف والضغط المطوّل وغيرها تعمل على الجوال بشكل مثالي لكن على Hyprland لا تفعل شيئاً. هذه المقالة تشرح لماذا وكيف تحلّها.

لماذا لا تعمل؟ — المشكلة الجذرية

هناك ثلاث طبقات معنية:

  • الطبقة الأولى — السماعة: حين تضغط الزر، السماعة ترسل إشارة بلوتوث باستخدام بروتوكول AVRCP (Audio Video Remote Control Profile). السماعة تقوم بعملها بشكل صحيح.
  • الطبقة الثانية — BlueZ: daemon البلوتوث على Linux يستقبل إشارة AVRCP ويفهم أن أمر تحكم بالميديا جاء — لكنه لا يعرف أين يرسله. من الـ Media Player؟ Spotify؟ Firefox؟ MPV؟ BlueZ يتعامل مع الاتصال فقط لا التشغيل.
  • الطبقة الثالثة — تطبيقات الميديا: كل Spotify وFirefox وMPV يتحدثون لغة MPRIS (Media Player Remote Interfacing Specification) — نفس اللغة التي يستخدمها playerctl.

السماعة تتحدث AVRCP، تطبيقات الميديا تتحدث MPRIS، ولا أحد يترجم بينهما. على GNOME وKDE هذه الترجمة موجودة في الـ settings daemon — لكن Hyprland لا يملك settings daemon، فالاتصال مقطوع.

الحل — mpris-proxy

البرنامج المسؤول عن الترجمة اسمه mpris-proxy ويأتي ضمن حزمة bluez-utils الموجودة غالباً مثبّتة مسبقاً: 

# تثبيت في حال لم يكن موجوداً
sudo pacman -S bluez-utils

# تشغيله يدوياً للتجربة
mpris-proxy

إذا رأيت رسالة مثل org.bluez appeared, Bluetooth adapter found فهو يعمل بشكل صحيح. جرّب الضغط على زر السماعة الآن — يجب أن يعمل التشغيل/الإيقاف.

تشغيله تلقائياً عند الدخول

أضفه في ملف الـ autostart:

# في hyprland.conf أو autostart.conf
exec-once = mpris-proxy
💡 لا تحتاج لإضافة & أو disown هنا — exec-once يشغّل البرنامج في الخلفية تلقائياً.

التحقق من أن كل شيء يعمل

# تأكد أن mpris-proxy يعمل
pgrep -x mpris-proxy && echo "يعمل ✓" || echo "لا يعمل ✗"

# إعادة تشغيله إذا توقف
pkill mpris-proxy; mpris-proxy &

الملخص

إيماءات التراك باد مدمجة في Hyprland وتعمل بدون برامج إضافية — فقط أضف سطور gesture في ملف الإعداد. ابدأ بالثلاثة الأساسية (تبديل workspace، إغلاق، fullscreen) ثم أضف حسب احتياجك.

إيماءات البلوتوث تحتاج فقط تشغيل mpris-proxy في الـ autostart — سطر واحد يحل المشكلة كاملاً.

🎯 كلا الحلّين بسيطان جداً حين تفهم السبب — وهذا هو الفرق بين من ينسخ ويلصق ومن يفهم ما يفعله.
