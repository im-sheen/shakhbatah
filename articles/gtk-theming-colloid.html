الطريقة الوحيدة التي تنجح فعلاً لتثيم GTK على Hyprland

كيف تصنع ثيم GTK3 وGTK4 بألوانك الخاصة تماماً باستخدام Colloid وSASS — بدون Oomox أو تعديل CSS مباشر.

١٧ يناير ٢٠٢٦

المشكلة — الـ Boss الأخير في الـ Ricing

قضيت ساعات في ضبط الـ rice. الـ bar مثالي، ألوان الترمنل متناسقة، كل شيء يتطابق. ثم تفتح Thunar — مدير الملفات — فيبدو كأنه من جهاز آخر تماماً.

تثيم GTK هو الـ final boss في رحلة الـ ricing. معظم الناس يختارون أقرب ثيم موجود ويتعايشون مع عدم التطابق. المقالة دي هتريك كيف تصنع ثيم GTK3 وGTK4 بألوانك الدقيقة في حوالي عشر دقايق.

💡 النتيجة: تطبيقات GTK3 مثل Calculator، وGTK4 مثل Clocks وNautilus — كلها بنفس لوحة الألوان بالضبط.

لماذا الطرق الشائعة تفشل

تثيم GTK معروف بأنه فوضوي تاريخياً، وأسباب ذلك واضحة:

  • ثيمات GTK3 لا تعمل بالكامل على GTK4
  • تطبيقات libadwaita تتجاهل معظم الثيمات كلياً
  • تجد ثيماً يعجبك لكن لون الـ accent خطأ — تحاول تعديل CSS مباشرة فتختفي الأزرار وتتكسر الـ checkboxes

معظم الناس يجربون أحد خيارين:

١. Oomox / Themix

أدوات قديمة والناتج منها خشن. والأسوأ أنها لا تدعم libadwaita أو GTK4 بشكل صحيح.

٢. تعديل ثيم Compiled مباشرة

تدخل مجلد الثيم وتغير قيم hex في CSS — يشتغل جزئياً حتى تختلف الـ metadata مع الألوان، فيظهر الثيم باسم مختلف عن الألوان اللي اخترتها، ويصبح التنقل بين الثيمات فوضى.

✅ الحل الصحيح: تعدّل الثيم على مستوى الـ source قبل الـ compile — فكل شيء يبقى متسق، وعملية البناء تتولى كل حالات الحافة، وتحصل على ثيم يشتغل فعلاً.

الأداة: Colloid GTK Theme

الثيم الأساسي اللي سنستخدمه هو Colloid GTK Theme من المطور Vince Lewis. اخترناه لأسباب محددة:

  • يُحدَّث باستمرار — آخر تغييرات الشهر الماضي وليس قبل سنتين
  • يدعم GTK3 وGTK4 وlibadwaita
  • مصمم بطريقة تجعل إضافة لوحات ألوان جديدة سهلة جداً

بديل مناسب هو Graphite من نفس المطور إن أردت مظهراً أكثر إحكاماً، لكن تحذير: ثيمات أخرى قد لا تكون مُعدَّة لهذا الـ workflow بالضبط.

المتطلبات

قبل أي شيء، ثبّت هذه الحزم:

# sass-c ضروري — الثيم لن يُبنى بدونه
sudo pacman -S sassc

# اختياري: لو ستستخدم GTK2
sudo pacman -S gtk-engine-murrine

الخطوة ١ — Clone الـ Repo

git clone https://github.com/vinceliuice/Colloid-gtk-theme.git
cd Colloid-gtk-theme

بعد دخولك للمجلد ستجد بنية واضحة. المجلد المهم هو src/sass/ — هنا تسكن ملفات لوحات الألوان.

الخطوة ٢ — قرر لوحة ألوانك أولاً

قبل أي كود، احتاج تحدد ألوانك. على الأقل تحتاج 16 لوناً: لون خلفية، لون نص، لون accent، وعدة درجات بينها للـ hover states والـ borders.

إذا كنت تبني ثيماً موجوداً (مثل Horizon أو Catppuccin)، ابحث عن ملف الألوان الرسمي. أما ألوان الخلفية المتدرجة (shades) فهذا الموقع يوفرها بشكل مباشر:

🌐 اذهب إلى colorhexa.com وادخل لون الخلفية الأساسي لديك. ستجد قسم Shades وقسم Tints — الـ shade يضيف أسود للون، الـ tint يضيف أبيض. اختر التدرجات من 050 إلى 900 واحفظها.

مثال: لو لون خلفيتك هو Horizon (#1C1E26):

# ادخل هذا على colorhexa.com
1C1E26

# ستحصل على تدرجات مثل:
# 050 → أغمق درجة (تقريباً أسود)
# 900 → أفتح درجة (تقريباً أبيض)
# اختر ما يتناسب مع لوحتك

الخطوة ٣ — أنشئ ملف لوحة الألوان

cd src/sass

# انسخ ملف موجود كقاعدة (Cappuccino مثلاً)
cp _color-palette-cappuccin.scss _color-palette-horizon.scss

افتح الملف الجديد وعدّل الألوان:

// _color-palette-horizon.scss

// اسم اللوحة
$scheme-name: 'Horizon Color Palette';

// الألوان الأساسية (الـ 16 terminal colors)
$red:        #E95678;
$red-light:  #F09383;
$green:      #29D398;
$green-light:#27D796;
$yellow:     #FAB795;
$blue:       #26BBD9;
$blue-light: #59E3E3;
$magenta:    #EE64AC;
$cyan:       #59E3E3;
$cyan-light: #59E3E3;

// درجات الخلفية (من colorhexa.com)
$bg-050:  #141620;
$bg-100:  #16181E;
$bg-150:  #1A1C22;
$bg-200:  #1C1E26;   // ← لون الخلفية الأساسي
$bg-300:  #232530;
$bg-400:  #2E303E;
$bg-500:  #3D3F4E;
// ... وهكذا حتى bg-900

// الأبيض والأسود
$white:   #D5D8DA;   // foreground color
$black:   #1C1E26;   // background color
💡 لوحات الألوان الجاهزة مثل Catppuccin وDracula وEverforest موجودة بالفعل في الـ repo — ابحث في src/sass/ عن اسمها.

الخطوة ٤ — سجّل اللوحة في install.sh

إنشاء الملف وحده لا يكفي — يجب إخبار سكربت البناء بوجوده. افتح install.sh في جذر الـ repo وأضف اسمك في ثلاثة أماكن:

المكان الأول — scheme_variants

# ابحث عن هذا السطر وأضف اسمك
scheme_variants=("default" "rayan" "horizon")
#                                   ↑ أضفه هنا

المكان الثاني — نص المساعدة (usage)

# داخل قسم --tweaks في نص help
# أضف:
horizon

المكان الثالث — case statement في دالة color_schemes

# انسخ block موجود مثل cappuccin وعدّله:
horizon)
  scheme_color='horizon'
  color_scheme="$scheme_color"
  ;;

للتحقق أن كل شيء صحيح:

./install.sh --help
# يجب أن ترى "horizon" ضمن الخيارات المتاحة

الخطوة ٥ — ابنِ الثيم

# بناء النسخة الداكنة من ثيم Horizon
./install.sh --tweaks horizon --color dark

# بناء النسخة الفاتحة أيضاً إن أردت
./install.sh --tweaks horizon --color light

بعد اكتمال البناء، ستجد مجلد الثيم في ~/.themes/.

الخطوة ٦ — فعّل الثيم

GTK3

الثيم يُفعَّل تلقائياً من ~/.themes/ عبر أي أداة مثل nwg-look أو gsettings:

gsettings set org.gnome.desktop.interface gtk-theme 'Colloid-Dark-Horizon'

GTK4 وlibadwaita

هذا الجزء يختلف — GTK4 يقرأ من مكان مختلف:

# انسخ مجلد GTK4 إلى config
cp -r ~/.themes/Colloid-Dark-Horizon/gtk-4.0 ~/.config/gtk-4.0

# تأكد من وجود هذين الملفين
ls ~/.config/gtk-4.0/
# gtk.css  gtk-dark.css
⚠️ لا تلمس أي إعداد آخر في مجلد GTK4. هذان الملفان فقط هما المطلوبان لتطبيقات libadwaita.

أخطاء شائعة تجنّبها

❌ لا تستخدم Oomox أو Themix

أدوات قديمة والناتج منها يحتوي عناصر مكسورة. لا تدعم GTK4 الحديثة. هذه الطريقة أكثر يدوية لكن الناتج يُستخدم فعلاً.

❌ لا تعدّل ثيماً Compiled

تغيير hex values داخل CSS الثيم يسبب عدم تطابق الـ metadata مع الألوان. ابنِ من الـ source دائماً عبر SASSC — هذا هو الفرق.

✅ إذا ظهر شيء خطأ بعد البناء

في الغالب لون واحد خاطئ في ملف اللوحة. تحقق من الـ shades والـ accent color، أعد البناء، وجرب مجدداً. نادراً تحتاج أكثر من ٢-٣ تكرارات.

أكثر من ثيم — بنفس الطريقة

هذه الطريقة قابلة للتوسع. تريد ثيماً داكناً وآخر فاتحاً؟ أو مجموعة كاملة لكل موسم؟ كرر نفس العملية:

  1. أنشئ ملف _color-palette-YOURNAME.scss جديد
  2. سجّله في install.sh
  3. ابنِه باسم مختلف
# مثال: بناء ثيمات متعددة
./install.sh --tweaks horizon  --color dark
./install.sh --tweaks catppuccin --color dark
./install.sh --tweaks dracula  --color dark

يمكنك بعدها ربطها بـ theme switcher يغير الثيم مع الخلفية تلقائياً — وهذا موضوع مقالة منفصلة.

خلاصة

العملية بالكامل في ٦ خطوات:

  1. ثبّت sassc وgtk-engine-murrine
  2. Clone مستودع Colloid GTK Theme
  3. حدد 16 لوناً للوحتك (استخدم colorhexa.com للتدرجات)
  4. أنشئ _color-palette-YOURNAME.scss في src/sass/
  5. سجّل اللوحة في install.sh في ثلاثة أماكن
  6. ابنِ بـ ./install.sh --tweaks YOURNAME --color dark وانسخ GTK4 لـ ~/.config/gtk-4.0
🎨 النتيجة النهائية: تطبيقاتك GTK3 وGTK4 كلها بنفس لوحة الألوان بالضبط — بدون حلول وسط.
