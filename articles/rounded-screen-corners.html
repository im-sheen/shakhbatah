زوايا الشاشة المدوّرة في Hyprland باستخدام AGS

كيف تضيف زوايا مدوّرة لشاشتك في Hyprland باستخدام AGS وTypeScript — شرح كامل للمنطق والكود.

١٦ مايو ٢٠٢٦

مقدمة

إضافة زوايا مدوّرة لحواف الشاشة هي واحدة من أبسط التفاصيل التي ترفع مستوى الـ rice وتعطيه طابعاً احترافياً. الفكرة بسيطة: نرسم مربعاً صغيراً في كل زاوية من الشاشة، ثم نقطع منه دائرة لتظهر الاستدارة. الأداة التي سنستخدمها هي AGS (Aylur's GTK Shell) مع TypeScript وكتابة أكواد Cairo لرسم الزوايا.

💡 AGS هو shell سطح المكتب الذي يتيح لك بناء أشرطة وـ overlays وغيرها باستخدام TypeScript/JSX. سنستخدمه هنا لشيء واحد فقط: رسم زوايا الشاشة.

الخطوة الأولى: تثبيت AGS

على Arch Linux، ابحث عن الحزمة باسم aylurs-gtk-shell من الـ AUR أو الـ Chaotic AUR: 

# من الـ AUR
yay -S aylurs-gtk-shell

# أو من Chaotic AUR (إذا كان مفعّلاً)
sudo pacman -S aylurs-gtk-shell-git

بعد التثبيت، ثبّت أيضاً مكتبات libastal الكاملة لضمان توافر كل المكوّنات: 

sudo pacman -S libastal-meta
📦 libastal-meta هي meta package تثبّت كل مكتبات libastal مرّة واحدة (wireplumber، power-profiles، وغيرها) — مفيد خصوصاً إذا كنت تخطط لاستخدام AGS لمشاريع أخرى لاحقاً.

الخطوة الثانية: إنشاء مجلد المشروع

أنشئ مجلداً جديداً للمشروع وشغّل ags init بداخله: 

mkdir ~/corners && cd ~/corners
ags init -d . -f

الخيار -d . يحدد المجلد الحالي، و-f يتجاوز أي إعدادات سابقة موجودة. بعد انتهاء الأمر ستجد الهيكل التالي: 

corners/
├── node_modules/
├── package.json
├── style.css
└── widget/
    └── bar.tsx

الخطوة الثالثة: إنشاء ملف الزوايا

أنشئ ملفاً جديداً داخل مجلد widget: 

touch widget/screen-corners.tsx

ثم افتح app.ts وأضف الاستيراد والاستدعاء: 

import screenCorners from "./widget/screen-corners.tsx";

// أضف هذا السطر داخل الدالة الرئيسية
screenCorners();

المنطق الهندسي: كيف تعمل الزاوية؟

قبل الكود، من المهم أن تفهم الفكرة الرياضية البسيطة وراءها:

  • نرسم مربعاً 30×30 بكسل يُوضع في إحدى زوايا الشاشة.
  • نرسم دائرة مركزها عند النقطة (30, 30) بنصف قطر 30.
  • باستخدام قاعدة even-odd في Cairo، الجزء المتقاطع بين المربع والدائرة يُحذف — فتظهر الزاوية المستديرة.
  • نلوّن الجزء المتبقي بنفس لون خلفية الشاشة (عادةً أسود) لإخفائه.
🎨 فكّر فيه هكذا: المربع هو قطعة ورق، والدائرة هي مقص دائري — نقطع الزاوية لتبدو مستديرة.

الكود الكامل لـ screen-corners.tsx

هذا هو الكود الكامل الذي تضعه في ملف widget/screen-corners.tsx: 

import { Astal, Gtk } from "ags/gtk4";
import Cairo from "cairo";

const size = 30; // حجم الزاوية بالبكسل — غيّره حسب ذوقك

const corners = {
  topLeft: {
    anchor: Astal.WindowAnchor.TOP | Astal.WindowAnchor.LEFT,
    cx: size,
    cy: size,
  },
  topRight: {
    anchor: Astal.WindowAnchor.TOP | Astal.WindowAnchor.RIGHT,
    cx: 0,
    cy: size,
  },
  bottomLeft: {
    anchor: Astal.WindowAnchor.BOTTOM | Astal.WindowAnchor.LEFT,
    cx: size,
    cy: 0,
  },
  bottomRight: {
    anchor: Astal.WindowAnchor.BOTTOM | Astal.WindowAnchor.RIGHT,
    cx: 0,
    cy: 0,
  },
};

export default function screenCorners() {
  return Object.entries(corners).map(([name, { anchor, cx, cy }]) => {
    const da = new Gtk.DrawingArea();
    da.set_content_width(size);
    da.set_content_height(size);

    da.set_draw_func((_self, cr: Cairo.Context, width: number, height: number) => {
      // ارسم المربع
      cr.rectangle(0, 0, width, height);

      // ارفع القلم لرسم الدائرة بشكل منفصل
      cr.new_sub_path();

      // ارسم الدائرة (cx, cy = المركز، size = نصف القطر)
      cr.arc(cx, cy, size, 0, 2 * Math.PI);

      // even-odd: يُظهر فقط التقاطع بين المربع والمنطقة خارج الدائرة
      cr.set_fill_rule(Cairo.FillRule.EVEN_ODD);

      // اللون الأسود الشفاف — غيّره ليطابق لون سطح مكتبك
      cr.set_source_rgba(0, 0, 0, 1);

      cr.fill();
    });

    return (
      
        {da}
      
    );
  });
}

شرح أجزاء الكود

الـ anchor

يحدد أين يُوضع مربع الزاوية على الشاشة. الرمز | يجمع اتجاهين معاً (مثلاً: أعلى + يسار = الزاوية العلوية اليسرى).

cx و cy

إحداثيات مركز الدائرة التي ترسمها Cairo. تختلف حسب الزاوية:

  • أعلى يسار: المركز عند (size, size) أي الزاوية السفلية اليمنى من المربع
  • أعلى يمين: المركز عند (0, size)
  • أسفل يسار: المركز عند (size, 0)
  • أسفل يمين: المركز عند (0, 0)

FillRule.EVEN_ODD

هذه القاعدة هي سر عمل الزوايا. بدلاً من ملء كل شيء داخل الشكل، Cairo يملأ فقط المناطق التي تُحاط بعدد فردي من المسارات. النتيجة: الدائرة تُحذف من المربع وتظهر الزاوية المستديرة.

exclusivity و layer

exclusivity={Astal.Exclusivity.IGNORE}
layer={Astal.Layer.OVERLAY}

IGNORE يعني أن Hyprland لن يحجز مساحة للزاوية (لن تؤثر على النوافذ). OVERLAY يضع الزاوية فوق كل شيء لتبدو كجزء من الشاشة نفسها.

تشغيل الزوايا

أوقف أي جلسة AGS حالية ثم شغّل المشروع:

ags quit
ags run app.ts

للتحقق من أن الزوايا تعمل كـ layer، استخدم hyprctl:

hyprctl layers

ستجد في الناتج namespace: corner-topLeft وبقية الزوايا.

تلميحات ومشاكل شائعة

الزوايا تظهر بخلفية ملوّنة

تأكد أن خاصية css تحتوي على background-color: transparent;. بدونها ستظهر خلفية رمادية أو بيضاء حول الزاوية. 

css="background-color: transparent;"

تغيير حجم الزاوية

غيّر قيمة size في بداية الملف. القيمة الافتراضية 30 مناسبة لمعظم الشاشات. ارفعها إلى 50 أو أكثر إذا أردت زوايا أكبر وأوضح.

تغيير اللون

إذا كانت خلفية سطح مكتبك ليست سوداء، غيّر set_source_rgba: 

// أبيض
cr.set_source_rgba(1, 1, 1, 1);

// لون مخصص (R, G, B, Alpha)
cr.set_source_rgba(0.1, 0.05, 0.15, 1);

التشغيل التلقائي مع Hyprland

أضف السطر التالي في hyprland.conf ليبدأ AGS مع الجلسة:

exec-once = ags run ~/corners/app.ts

الخلاصة

بضعة أسطر من TypeScript وCairo وتحصل على زوايا مدوّرة حقيقية لشاشتك. الفكرة بسيطة: مربع + دائرة + even-odd = زاوية مستديرة. الجميل أنك تتحكم في كل شيء: الحجم، اللون، والموضع.

إذا أردت أن تجعل الزوايا تتغير مع الـ theme switcher تلقائياً، يمكنك ربط set_source_rgba بمتغير يقرأ لون الخلفية الحالي من ملف الإعدادات.
