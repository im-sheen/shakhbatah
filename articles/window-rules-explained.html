قواعد النوافذ في Hyprland — شرح كامل

كيف تتحكم في سلوك أي نافذة في Hyprland — من الصيغة الأساسية إلى الـ Tags والأخطاء الشائعة التي تضيّع وقتك.

١٦ مايو ٢٠٢٦

المشكلة

تثبّت Hyprland، تفتح المتصفح فيتبلّط. تفتح نافذة حوار صغيرة فتتبلّط هي الأخرى. كل شيء يُعامَل بنفس الطريقة. الحل؟ Window Rules — وهي واحدة من أقوى ميزات Hyprland، لكن معظم الناس يستخدمون 10% منها فقط.

النموذج الذهني: if-then

قاعدة النافذة هي جملة شرطية بسيطة: إذا تطابقت النافذة مع كذا، طبّق عليها كذا.

  • الـ Prop — شرط التطابق (من؟ أي نافذة؟)
  • الـ Effect — ما يُطبَّق عليها (عائم؟ شفافية؟ حجم معين؟)

قاعدة واحدة تحتاج prop واحد على الأقل وeffect واحد على الأقل. يمكن دمج effects متعددة في قاعدة واحدة وكلها تُطبَّق على أي نافذة تتطابق.

صيغتان لكتابة القواعد

هناك طريقتان مقبولتان تماماً:

١. الصيغة المسمّاة (Block)

windowrule {
    # اسم اختياري للقاعدة — مفيد للتفعيل/الإيقاف ديناميكياً
    float = on
    size  = 800 600
    match = class kitty
}

٢. الصيغة المختصرة (One-liner)

windowrule = float on, size 800 600, match:class kitty

النتيجة واحدة تماماً. الصيغة المسمّاة أوضح عند وجود effects كثيرة، والمختصرة أسرع للقواعد البسيطة.

💡 فرق مهم: القواعد المسمّاة يمكن تفعيلها/إيقافها وقت التشغيل عبر hyprctl dispatch. القواعد المختصرة لا يمكن التحكم فيها ديناميكياً. إذا كنت تخطط لربط القاعدة بـ keybind، امنحها اسماً.

كيف تعرف class النافذة؟

الأمر المهم الذي ستستخدمه كثيراً:

hyprctl clients

يُخرج قائمة بكل النوافذ المفتوحة مع جميع خصائصها — الـ class، الـ title، الـ initial class وغيرها. ابحث فيها عن النافذة التي تريد استهدافها.

⚠️ لا تخمّن اسم الـ class. مثلاً class برنامج Spotify ليس بالضرورة spotify — قد يكون Spotify بحرف كبير أو غير ذلك. دائماً تحقق عبر hyprctl clients.

الـ Props — كيف تستهدف النوافذ

أهم الـ props التي ستحتاجها:

الـ Propما يستهدفه
match:class kittyالنوافذ التي class-ها يحتوي على "kitty"
match:title YouTubeالنوافذ التي عنوانها يحتوي على "YouTube"
match:float trueكل النوافذ العائمة
match:fullscreen trueالنوافذ في وضع fullscreen
match:xwayland trueتطبيقات Xwayland
match:workspace 3النوافذ في الـ workspace رقم 3
match:focus trueالنافذة النشطة حالياً

الـ class والـ title يدعمان Regex. تطابق جزئي افتراضي، وللتطابق الكامل استخدم الأقواس: 

# يتطابق مع أي class يحتوي على "kitty"
match:class kitty

# يتطابق فقط مع class اسمه kitty بالضبط
match:class (^kitty$)

# يتطابق مع أي class ما عدا kitty
match:class (^(?!kitty).*$)

عند دمج props متعددة في قاعدة واحدة، كلها يجب أن تتحقق (AND وليس OR): 

# تُطبَّق فقط على نوافذ kitty العائمة — الاثنان معاً
windowrule = opacity 0.85, match:class kitty, match:float true

Effects ثابتة vs ديناميكية

هذا الفرق هو سبب 90% من مشاكل "قاعدتي لا تعمل":

Effects الثابتة (Static)

تُطبَّق مرة واحدة فقط عند فتح النافذة ولا تُعاد تقييمها. أمثلة: float، tile، move، size، center، workspace، monitor، pin.

لأنها تعمل مرة واحدة، فهي تتطابق مع الـ initial class والـ initial title فقط. إذا غيّر المتصفح عنوانه بعد الفتح، القاعدة الثابتة لن تُعاد.

Effects الديناميكية (Dynamic)

تُعاد تقييمها في كل مرة يتغير فيها أي خاصية ذات صلة. أمثلة: opacity، bordercolor، rounding، bordersize، noblur، noshadow، animation.

مثال عملي على الفرق:

# border أحمر للنوافذ العائمة، أزرق للمثبّتة — يعمل لأن bordercolor ديناميكي
windowrule = bordercolor rgb(FF0000), match:class kitty, match:float true
windowrule = bordercolor rgb(0000FF), match:class kitty, match:float false

عند تبديل النافذة بين عائمة ومثبّتة، لون الـ border يتغير فوراً. هذا مستحيل مع effects ثابتة.

ترتيب القواعد والأولوية

القواعد تُعالَج من الأعلى للأسفل، وآخر قاعدة متطابقة تفوز. الترتيب مهم جداً: 

# كل نوافذ kitty: شفافية 0.8
windowrule = opacity 0.8, match:class kitty

# kitty العائمة تحديداً: شفافية 0.5 (تتغلب على السابقة)
windowrule = opacity 0.5, match:class kitty, match:float true

إذا عكست الترتيب، kitty ستحصل دائماً على 0.8 حتى وهي عائمة.

💡 القواعد المسمّاة تأخذ أولوية على المختصرة بغض النظر عن موضعها في الكونفيج — المسمّاة تُقيَّم أولاً دائماً.

أمثلة عملية جاهزة للاستخدام

جعل نوافذ الحوار عائمة ومتمركزة

windowrule = float on, center on, match:modal true

Firefox Picture-in-Picture عائمة ومثبّتة

windowrule {
    match = class firefox
    match = title Picture-in-Picture
    float = on
    pin   = on
    size  = 640 360
    # move بالنسبة لحافة الشاشة — بدون مسافات حول العمليات الحسابية
    move  = monitor_w-660 20
}

شفافية مع استثناء fullscreen

# active opacity, inactive opacity, fullscreen opacity
windowrule = opacity 0.9 override 0.85 override 1.0 override, match:class (^.*$)

كلمة override تجعل القيمة مطلقة لا مضاعِفة. بدونها تتضاعف قيم الشفافية وتحصل على نتائج غير متوقعة.

تعطيل الـ blur لتطبيق ثقيل

windowrule = noblur on, match:class firefox

فتح Discord في workspace 4 بصمت

windowrule = workspace 4 silent, match:class discord

silent تعني: انقل النافذة للـ workspace لكن لا تنتقل إليه.

الـ Tags — حيث تصبح القواعد أقوى

الـ Tag هو تسمية تُلصقها بأي نافذة، ثم تكتب قواعد تستهدف هذه التسمية بدلاً من الـ class. هذا يفتح إمكانيات مذهلة.

مثال: تطبيق شفافية 20% على أي نافذة بضغطة مفتاح:

# keybind: ألصق tag "alpha" على النافذة الحالية
bind = SUPER CTRL, Z, tagwindow, alpha

# قاعدة: أي نافذة عليها tag "alpha" تصبح شفافة
windowrule = opacity 0.2 override, match:tag alpha

اضغط Super+Ctrl+Z على أي نافذة فتصبح شبه شفافة. اضغطها مرة أخرى والـ tag يُزال والشفافية تعود.

Tags لتوحيد قواعد Terminal

# ألصق tag "term" على كل المحاكيات
windowrule = tag term, match:class (^kitty$)
windowrule = tag term, match:class (^foot$)
windowrule = tag term, match:class (^alacritty$)

# قاعدة واحدة تسري على الكل
windowrule = opacity 0.9, match:tag term

إذا غيّرت المحاكي لاحقاً، تُعدِّل سطر الـ tag فقط — لا تُعدِّل عشرين قاعدة منفصلة.

الأخطاء الشائعة

١. Static effect مع عنوان يتغير

# ❌ لن يعمل — بحلول وقت ظهور "YouTube" في العنوان، القواعد الثابتة انتهت
windowrule = float on, match:title YouTube

# ✅ استخدم effect ديناميكي بدلاً من ذلك
windowrule = opacity 0.95, match:title YouTube

٢. الشفافية تتضاعف

# ❌ إذا كان لديك opacity عام 0.5، هذا يعطي 0.5 × 0.5 = 0.25
windowrule = opacity 0.5, match:class kitty

# ✅ استخدم override للقيمة المطلقة
windowrule = opacity 0.5 override, match:class kitty

٣. تخمين اسم الـ class

دائماً استخدم hyprctl clients للتحقق من الاسم الفعلي. لا تخمن — أنت ستضيع وقتك في debug قاعدة صحيحة الصياغة لكن خاطئة الهدف.

خلاصة سريعة

  • Prop = من تستهدف | Effect = ماذا تفعل
  • hyprctl clients — دائماً للتحقق من الـ class
  • Static effects تعمل مرة عند الفتح، Dynamic تتابع التغييرات
  • آخر قاعدة متطابقة تفوز — الترتيب مهم
  • القواعد المسمّاة تتغلب على المختصرة وتدعم التفعيل الديناميكي
  • Tags = قوة حقيقية — استخدمها لتوحيد القواعد وتبسيط الكونفيج
  • override في الـ opacity = قيمة مطلقة لا مضاعِفة
