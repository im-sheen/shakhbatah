️ تخطيط Master — دليل شامل

شرح إعداد تخطيط Master — Master/Slave ratio والـ new_status وترتيب النوافذ وجميع الخيارات.

١٦ مايو ٢٠٢٦

ما هو تخطيط Master؟

في تخطيط Master، لديك نافذة رئيسية (master) تحتل معظم الشاشة، وبقية النوافذ (slaves) تتراص على الجانب الآخر. هذا هو تخطيط DWM الكلاسيكي.

تفعيل تخطيط Master

# في hyprland.conf
general {
    layout = master
}

الخيارات الأساسية

master {
    # نسبة Master/Slaves (0.5 = 50/50)
    mfact = 0.55

    # وضع النوافذ الجديدة: slave, master, inherit
    new_status = slave

    # نوافذ جديدة فوق باقي الـ slaves
    new_on_top = false

    # ترتيب النافذة الجديدة relative للنافذة المفعّلة
    new_on_active = after    # none/before/after

    # حجم النوافذ في Scratchpad
    special_scale_factor = 0.8

    # لا تكبّر نافذة وحيدة لتملأ Slave area
    always_keep_position = false

    # إعادة ترتيب أذكى عند Resize
    smart_resizing = true

    # سقوط النوافذ في مكان المؤشر
    drop_at_cursor = true
}

إضافة Master بـ keybind

# إضافة/إزالة master
bind = SUPER, M, layoutmsg, swapwithmaster
bind = SUPER SHIFT, M, layoutmsg, addmaster

new_status — أين تولد النوافذ؟

  • slave — كل نافذة جديدة تولد كـ slave (سلوك DWM الكلاسيكي)
  • master — كل نافذة جديدة تصبح master وتُدفع السابقة لـ slave
  • inherit — النافذة الجديدة تأخذ وضع النافذة المفعّلة حالياً

new_on_active — ترتيب بالنسبة للنافذة المفعّلة

  • none — لا اعتبار للنافذة المفعّلة
  • before — تُضاف قبل النافذة المفعّلة
  • after — تُضاف بعد النافذة المفعّلة (الأكثر طبيعية)
