Window Rules في Hyprland — الدليل الكامل

كيف تتحكم في سلوك كل نافذة — من الصيغة والـ props حتى الـ tags والأمثلة العملية الجاهزة للنسخ.

١٦ مايو ٢٠٢٦

المفهوم الأساسي

تُثبّت Hyprland وتفتح المتصفح فيتحول إلى نافذة tiled. تفتح نافذة حوار صغيرة فتُبلَّط هي الأخرى. كل شيء يُبلَّط والحل هو Window Rules — واحدة من أقوى ميزات Hyprland.

كل window rule هي جملة شرطية بسيطة: إذا تطابقت النافذة مع هذه الخصائص، طبّق عليها هذه التأثيرات. القسم الأول يُسمى prop (ما الذي تستهدف؟) والقسم الثاني يُسمى effect (ماذا تفعل؟).

صيغتان للكتابة

هناك طريقتان متكافئتان لكتابة window rules:

١. الصيغة الكتلية (Block Syntax) — الأوضح للقراءة

windowrule {
    match = class:kitty
    float = on
    size = 800 600
}

٢. الصيغة المختصرة (One-liner) — أسرع للقواعد البسيطة

windowrule = float on, size 800 600, match:class:kitty
💡 فرق مهم: القواعد المسمّاة (block syntax) يمكن تفعيلها وتعطيلها ديناميكياً عبر hyprctl في وقت التشغيل. القواعد المختصرة لا يمكن التحكم فيها بهذه الطريقة — إذا كنت تخطط لتبديل قاعدة بضغطة مفتاح، أعطِها اسماً.

Props — كيف تستهدف نافذة معينة

كل prop يبدأ بـ match ويُحدد أي النوافذ تستهدفها. أكثرها استخداماً:

  • match:class:kitty — يستهدف النوافذ التي class تحتوي على "kitty"
  • match:title:YouTube — يستهدف النوافذ التي عنوانها يحتوي على "YouTube"
  • match:float:true — يستهدف النوافذ العائمة
  • match:fullscreen:true — يستهدف النوافذ المملوءة للشاشة
  • match:xwayland:true — يستهدف تطبيقات XWayland
  • match:workspace:3 — يستهدف النوافذ في workspace رقم 3
  • match:focus:true — يستهدف النافذة المُركَّز عليها حالياً
🔍 لمعرفة class وعنوان أي نافذة مفتوحة، نفّذ في الطرفية:
hyprctl clients

Regex في المطابقة

الـ props تستخدم Regex للمطابقة. match:class:kitty يطابق أي class تحتوي كلمة "kitty". للمطابقة الدقيقة: 

# مطابقة دقيقة تماماً
match:class:(^kitty$)

# يطابق كل شيء ماعدا kitty
match:class:negative:(kitty)

عند وضع أكثر من prop في نفس القاعدة، يجب أن تتحقق جميعها معاً (AND وليس OR):

# تنطبق فقط على نوافذ kitty العائمة
windowrule {
    match = class:kitty
    match = float:true
    opacity = 0.85
}

Effects — ما الذي تفعله بالنافذة

هناك نوعان من التأثيرات، والفرق بينهما حاسم:

Static Effects — تُطبَّق مرة واحدة عند فتح النافذة

تشمل: float، tile، move، size، center، workspace، monitor، pin، fullscreen

⚠️ Static effects تعمل على class وعنوان النافذة وقت الفتح فقط. لو تغيّر عنوان المتصفح بعد فتحه (مثل تغيير التبويب)، لن تُعيد هذه القواعد تقييم نفسها.

Dynamic Effects — تُعاد تقييمها كلما تغيّرت خصائص النافذة

تشمل: opacity، bordercolor، rounding، bordersize، noblur، noshadow، animation، idleinhibit، tag

مثال عملي: لون إطار مختلف للنوافذ العائمة والمُبلَّطة:

# إطار أحمر للنوافذ العائمة
windowrule = bordercolor rgb(FF0000), match:class:kitty, match:float:true

# إطار أزرق للنوافذ المُبلَّطة
windowrule = bordercolor rgb(0000FF), match:class:kitty, match:float:false

لأن bordercolor ديناميكي، سيتغير اللون فوراً حين تُعوّم النافذة أو تُعيدها للتبليط.

ترتيب القواعد وأولويتها

القواعد تُعالَج من أعلى إلى أسفل، وآخر قاعدة مطابقة تفوز: 

# كل نوافذ kitty تحصل على شفافية 0.8
windowrule = opacity 0.8, match:class:kitty

# نوافذ kitty العائمة تحصل على 0.5 (تُلغي القاعدة أعلاه)
windowrule = opacity 0.5, match:class:kitty, match:float:true

إذا عكست الترتيب، ستحصل نوافذ kitty دائماً على 0.8 حتى وهي عائمة. نفس القواعد، نتائج مختلفة كلياً.

📌 القواعد المسمّاة (block syntax) تُقيَّم قبل القواعد المختصرة دائماً، بغض النظر عن موضعها في الملف.

أمثلة عملية جاهزة للنسخ

تعويم نوافذ الحوار وتوسيطها

windowrule = float on, center on, match:modal:true

Firefox Picture-in-Picture — عائمة ومثبّتة في الركن

windowrule {
    match = class:firefox
    match = title:Picture-in-Picture
    float = on
    pin = on
    size = 640 360
    move = onscreen 100%-660 20
}

الشفافية مع استثناء وضع ملء الشاشة

# active opacity, inactive opacity, fullscreen opacity
windowrule = opacity 0.9 override 0.85 override 1.0 override, match:class:.*
💡 كلمة override تجعل القيمة مطلقة بدلاً من مضاعف. بدونها تتضاعف قيم الشفافية مع بعضها وقد تحصل على نوافذ شفافة أكثر مما تريد.

تعطيل Blur على تطبيق ثقيل

windowrule = noblur on, match:class:firefox

فتح Discord في workspace 4 بصمت

windowrule = workspace 4 silent, match:class:discord

تعيين نافذة لشاشة محددة

windowrule = monitor DP-1, match:class:obsidian

Tags — القوة الحقيقية لـ Window Rules

Tags هي تسميات تُلصقها بالنوافذ، ثم تكتب قواعد تستهدف التسمية لا التطبيق. هذا يُبسّط إدارة القواعد بشكل كبير.

مثال: زر يجعل أي نافذة شبه شفافة:

# في hyprland.conf — اضغط Super+Ctrl+Z لتلصق tag على النافذة الحالية
bind = Super Ctrl, Z, tagwindow, alpha_low

# قاعدة تستهدف النوافذ التي عليها هذا الـ tag
windowrule = opacity 0.2 override, match:tag:alpha_low

فائدة أكبر — تجميع كل الطرفيات تحت tag واحد:

# إلصاق tag "term" على كل الطرفيات
windowrule = tag term, match:class:(kitty|foot|alacritty|wezterm)

# الآن يكفي قاعدة واحدة تستهدف term بدل قاعدة لكل طرفية
windowrule = opacity 0.95, match:tag:term

أخطاء شائعة وكيف تتجنبها

الخطأ الأول: static rule مع عنوان يتغير

windowrule = float on, match:title:YouTube لن تعمل. حين تنفتح نافذة المتصفح، عنوانها ليس "YouTube" بعد. استخدم dynamic effect كـ opacity أو bordercolor إذا أردت سلوكاً يعتمد على عنوان يتغير.

الخطأ الثاني: تضاعف الشفافية

إذا ضبطت active_opacity = 0.5 في إعداداتك العامة ثم أضفت windowrule = opacity 0.5، النتيجة 0.25 — شبه غير مرئية. استخدم override دائماً في window rules.

الخطأ الثالث: تخمين اسم الـ class

class لـ Spotify ليست "spotify" الصغيرة بالضرورة. تحقق دائماً: 

hyprctl clients | grep -A5 "class:"
