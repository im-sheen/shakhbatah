تخطيط Dwindle — الدليل الشامل

شرح كل خيارات تخطيط Dwindle — من Pseudotiling وForce Split حتى Smart Split وضبط نسبة النوافذ.

١٦ مايو ٢٠٢٦

ما هو Dwindle؟

Dwindle هو التخطيط الافتراضي في Hyprland — مستوحى من BSPWM. كل نافذة جديدة تُقسّم المساحة المتاحة للنصف، مما ينتج شكلاً يشبه أوراق شجرة ثنائية.

الخيارات الأساسية

# في hyprland.conf
dwindle {
    # تتبع وضع الـ split — لا تغيّره عند Resize
    preserve_split = true    # موصى به دائماً

    # تقسيم ثابت على اليمين دائماً
    force_split = 2          # 0=cursor، 1=يسار، 2=يمين

    # نسبة التقسيم الافتراضية
    default_split_ratio = 1.0  # 1.0 = 50/50

    # خاصية Pseudotiling
    pseudotile = false       # يُفعَّل بـ keybind
}

Pseudotiling — تبليط النوافذ العائمة

يجعل نوافذ floating تتبع نظام التبليط بدلاً من التحرك الحر:

bind = SUPER, P, pseudo

Force Split — التحكم في اتجاه الـ Split

  • 0 — يتبع موضع المؤشر
  • 1 — دائماً لليسار/أعلى
  • 2 — دائماً لليمين/أسفل (سلوك BSPWM)

Smart Split

dwindle {
    smart_split = false  # إيقاف (موصى به)
}

عند تشغيله، موضع المؤشر يحدد اتجاه الـ split — مربك في الغالب.

Special Scale Factor — لـ Scratchpad

dwindle {
    special_scale_factor = 0.8  # 80% من الحجم الطبيعي
}

يصغّر النوافذ داخل الـ special workspace (scratchpad) مقارنة بالـ workspaces العادية.

Single Window Aspect Ratio

dwindle {
    # للشاشات العريضة — يمنع تمدد النافذة لكامل العرض
    single_window_aspect_ratio = 16 9
}

مفيد لشاشات 21:9 أو أعرض حيث نافذة واحدة تمتد بشكل غير مريح.
