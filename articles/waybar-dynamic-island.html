تحويل Waybar إلى Dynamic Island 🏝️

كيف تجعل شريط Waybar العادي يبدو مثل Dynamic Island من Apple — بدون أدوات إضافية، فقط CSS وبعض الإعدادات الذكية.

١٥ يناير ٢٠٢٤

الفكرة الأساسية

الشريط العادي في Waybar يمتد على كامل عرض الشاشة ويجلس على الحافة — هذا هو شكله الافتراضي. تأثير Dynamic Island يعتمد على ثلاثة أشياء فقط:

  • الشريط يطفو — لا يلتصق بحافة الشاشة
  • المحتوى في المنتصف — لا يمتد يميناً أو يساراً
  • شكل حبة دواء (Pill) — حواف مستديرة بالكامل

الحيلة هنا أنك لا تُصمّم الشريط نفسه، بل تُصمّم ما بداخله. الشريط يصبح شفافاً تماماً، والـ pill هو container الوحدة الوسطى module-center.

💡 ليس AGS، ليس EWW، ليس أي أداة خارجية — هذا Waybar العادي الذي يستخدمه الجميع.

١. إعداد config.json

أول شيء تحتاج ضبطه في ملف الإعداد:

الموضع والهامش

{
  "position": "top",
  "margin-top": 8,
  "margin-left": 0,
  "margin-right": 0
}

position: top يضع الشريط في الأعلى. margin-top يدفع الشريط للأسفل قليلاً بحيث لا يلتصق بحافة الشاشة — هذا هو ما يعطيه تأثير "الطفو".

وضع كل الـ Modules في المنتصف

{
  "modules-left": [],
  "modules-center": [
    "hyprland/workspaces",
    "clock",
    "battery",
    "network",
    "pulseaudio",
    "custom/media"
  ],
  "modules-right": []
}

هذه هي النقطة الجوهرية. عندما تضع كل شيء في modules-center، تتجمع العناصر في وسط الشاشة والجانبان يبقيان فارغَين. هذا بالضبط ما يخلق وهم الـ Dynamic Island المنفصل.

تعريف الـ Modules

{
  "hyprland/workspaces": {
    "format": "{icon}",
    "on-click": "activate"
  },

  "clock": {
    "format": "{:%H:%M}",
    "tooltip-format": "{:%A، %d %B %Y}"
  },

  "battery": {
    "format": "{icon} {capacity}%",
    "format-icons": ["󰁺", "󰁻", "󰁼", "󰁽", "󰁾", "󰁿", "󰂀", "󰂁", "󰂂", "󰁹"]
  },

  "network": {
    "format-wifi": "󰤨 {essid}",
    "format-disconnected": "󰤭 غير متصل",
    "tooltip-format": "{ipaddr}"
  },

  "pulseaudio": {
    "format": "{icon} {volume}%",
    "format-icons": {
      "default": ["󰕿", "󰖀", "󰕾"]
    },
    "on-click": "pavucontrol"
  }
}

٢. إعداد style.css

هنا يحدث الجمال الحقيقي. الـ CSS هو المسؤول الكامل عن تحويل الشريط العادي إلى Dynamic Island.

الخطوة الأولى: تصفير كل شيء

* {
  font-family: "JetBrainsMono Nerd Font", sans-serif;
  font-size: 14px;
  padding: 0;
  margin: 0;
  min-height: 0;
}

هذا يُصفّر كل القيم الافتراضية. فكّر فيه كـ main() في لغة C — نقطة البداية التي تضمن أن لا قيمة مسبقة تُفسد تصميمك.

الخطوة الثانية: جعل الشريط شفافاً

window#waybar {
  background: transparent;
  border: none;
  box-shadow: none;
}

إذا تركت للشريط لونه الافتراضي، ستحصل على شريط عادي يمتد على كامل الشاشة. الشفافية تجعل الشريط "يختفي" وتبقى فقط العناصر التي داخله مرئية.

الخطوة الثالثة: صنع الـ Pill — هنا السحر

.modules-center {
  background: #1e1e2e;
  padding: 4px 16px;
  border-radius: 999px;
  box-shadow: 0 4px 20px rgba(0, 0, 0, 0.4);
}

ننظر لكل سطر:

  • background — لون الـ pill، اختر أي لون يناسب ثيمك
  • padding — مسافة داخلية تعطي العناصر "تنفّساً"
  • border-radius: 999px — قيمة كبيرة جداً تجعل الحواف مستديرة بالكامل بغض النظر عن ارتفاع الشريط
  • box-shadow — اختياري لكن يبيع التأثير بقوة، يجعل الـ pill يبدو طافياً فوق سطح المكتب

الخطوة الرابعة: إزالة خلفيات الـ Modules الفردية

#workspaces,
#clock,
#battery,
#network,
#pulseaudio,
#custom-media {
  background: transparent;
  box-shadow: none;
  border: none;
  padding: 0 8px;
}

كل module بشكل افتراضي عنده خلفيته الخاصة. إذا تركتها، ستنكسر وهم الـ pill لأنك ستشوف مربعات داخل الـ pill. يجب أن تكون كل الـ modules شفافة حتى يظهر لون الـ pill بشكل موحّد.

🎨 بديل — نمط OnePlus Live Alerts: إذا أردت مظهراً مشابهاً لـ Live Alerts من OnePlus، أعطِ كل module خلفية بلون مختلف قليلاً عن الـ pill وأضف border خفيف. هذا يكسر وهم الـ pill الواحد لكن يعطي مظهراً آخر جميلاً.

٣. تصميم مساحات العمل (Workspaces)

المساحات تحتاج اهتماماً خاصاً لأن لها حالات متعددة: نشطة، غير نشطة، وعاجلة. 

/* الزر الافتراضي — غير نشط */
#workspaces button {
  background: transparent;
  color: #6c7086;
  border: none;
  border-radius: 999px;
  padding: 2px 10px;
  transition: all 0.2s ease;
}

/* عند التمرير */
#workspaces button:hover {
  background: rgba(255, 255, 255, 0.08);
  color: #cdd6f4;
}

/* مساحة العمل النشطة */
#workspaces button.active {
  background: linear-gradient(135deg, #cba6f7, #89b4fa);
  color: #1e1e2e;
  font-weight: bold;
}

/* مساحة عمل بها نوافذ طارئة */
#workspaces button.urgent {
  background: #f38ba8;
  color: #1e1e2e;
}

الـ transition مهم جداً. بدونه، يقفز زر المساحة النشطة فجأة عند التبديل. مع الـ transition، ينزلق بسلاسة.

الـ gradient في button.active اختياري تماماً — يمكنك استبداله بلون ثابت. لكن يبدو رائعاً خاصة مع ثيمات مثل Everforest أو Rosé Pine.

٤. تنسيق الـ Tooltips

التفصيلة الصغيرة التي تُكمل المظهر. عندما تمرر فوق module وتظهر tooltip بمظهر مختلف كلياً، تبدو الإعدادات غير محترفة. 

tooltip {
  background: #1e1e2e;
  border: 1px solid #313244;
  border-radius: 12px;
  padding: 6px 12px;
  color: #cdd6f4;
  font-size: 12px;
}

٥. تناسق أحجام الأيقونات

عند استخدام Nerd Fonts، أيقونة من Font Awesome قد تكون أكبر بكثير من أيقونة من Material Design — وهذا يكسر تناسق الـ pill. الحل يدوي: تضبط حجم كل أيقونة بعينها حتى تبدو متناسقة. 

/* مثال: تصغير أيقونة الشبكة قليلاً */
#network {
  font-size: 14px;
}

/* وتكبير أيقونة الصوت قليلاً */
#pulseaudio {
  font-size: 17px;
}

لا توجد قاعدة ثابتة هنا — استخدم عينيك وعدّل حتى تبدو كل الأيقونات بنفس الوزن البصري.

٦. نظام الألوان (اختياري لكن موصى به)

إذا كنت تريد إمكانية تبديل الثيمات لاحقاً، الأفضل فصل الألوان في ملف colors.css منفصل واستدعاؤه من style.css: 

/* في بداية style.css */
@import "colors.css";

ومحتوى colors.css يكون هكذا (مثال بثيم Rosé Pine):

/* rosepine.css */
@define-color bg0    #191724;
@define-color bg1    #1f1d2e;
@define-color bg2    #26233a;
@define-color bg3    #403d52;
@define-color bg4    #524f67;
@define-color fg     #e0def4;
@define-color gray1  #6e6a86;
@define-color accent #c4a7e7;

ثم تستخدمها في style.css هكذا:

.modules-center {
  background: @bg1;
}

#workspaces button.active {
  background: @accent;
  color: @bg0;
}

بهذه الطريقة، لتغيير الثيم كل ما تحتاجه هو استبدال ملف الألوان — كل الـ CSS يتكيف تلقائياً.

الملف الكامل — نظرة عامة

هيكل ملفات المشروع يجب أن يكون هكذا:

~/.config/waybar/
├── config.json      ← إعداد الـ modules والموضع
├── style.css        ← كل التصميم
└── themes/
    ├── rosepine.css
    ├── catppuccin.css
    └── everforest.css

لماذا يعمل هذا التصميم بشكل جيد؟

ثلاثة أسباب تجعل Dynamic Island أفضل من الشريط التقليدي:

  • مينيمالي: عنصر واحد في الشاشة بدل شريط يمتد على كل العرض. عيناك تتجه للمركز حيث المعلومات، لا للحواف.
  • طافٍ: الـ margin والـ shadow يخلقان عمقاً بصرياً. الشريط لا يبدو جزءاً من الشاشة — يبدو يطفو فوقها.
  • متماسك: كل شيء داخل pill واحد، خلفية واحدة، شكل واحد، ظل واحد. يبدو كجسم واحد لا كمجموعة widgets متفرقة.
🎯 الهدف ليس شريط معلومات — الهدف جزيرة صغيرة أنيقة تبقى بعيدة عن طريقك حتى تحتاجها.
