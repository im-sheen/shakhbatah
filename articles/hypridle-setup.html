Hypridle — إدارة الخمول في Hyprland

كيف تضبط Hypridle لإيقاف الشاشة وقفل الجلسة وتعليق النظام تلقائياً — شرح كامل مع config جاهز للنسخ.

١٦ مايو ٢٠٢٦

المشكلة: Hyprland لا يفعل شيئاً عند الغياب

بدون Hypridle، شاشتك تبقى مضاءة إلى الأبد ما إن تبتعد عن الجهاز. لا قفل تلقائي، لا إيقاف للشاشة، لا تعليق للنظام. إذا كنت على لابتوب فبطاريتك تستنزف بلا مبرر، وعلى الديسكتوب فأنت تهدر طاقة في كل مرة تبتعد.

Hypridle هو daemon إدارة الخمول الرسمي لـ Hyprland. يراقب غياب أي حركة (ماوس أو كيبورد)، ثم ينفّذ ما تأمره به — سواء كان إطفاء الشاشة أو قفل الجلسة أو تعليق النظام كلياً.

التثبيت

إذا كنت على Arch Linux مع Hyprland الرسمي: 

sudo pacman -S hypridle

وإذا كنت تستخدم نسخة hyprland-git، ستحتاج إلى النسخة المقابلة: 

yay -S hypridle-git
⚠️ تأكد من تطابق النسخة: إذا ثبّتت hyprland-git، استخدم hypridle-git أيضاً لتجنب مشاكل التوافق.

التشغيل التلقائي مع Hyprland

حتى يعمل Hypridle عند كل تسجيل دخول، أضف هذا السطر إلى ملف إعدادات Hyprland: 

exec-once = hypridle

إذا كنت تستخدم UWSM لإدارة الجلسة بدلاً من ذلك: 

systemctl --user enable --now hypridle.service

ملف الإعداد

Hypridle يقرأ إعداداته من: 

~/.config/hypr/hypridle.conf
📌 هذا الملف مطلوب — بدونه لن يعمل Hypridle.

بنية الملف تتكون من قسمين:

  • general — إعدادات عامة تسري على كل شيء (مثل أمر القفل وما يحدث قبل/بعد السكون)
  • listener — قواعد مُوقَّتة، كل واحدة تنتظر فترة خمول محددة ثم تنفّذ أمراً

شرح قسم general

general {
    lock_cmd = pidof hyprlock || hyprlock
    before_sleep_cmd = loginctl lock-session
    after_sleep_cmd = hyprctl dispatch dpms on
}

lock_cmd

هذا الأمر يُنفَّذ حين يريد Hypridle قفل الجلسة. السطر pidof hyprlock || hyprlock يعمل بذكاء: يتحقق أولاً إذا كان Hyprlock يعمل بالفعل، فإذا كان كذلك لا يفعل شيئاً، وإذا لم يكن يشغّله. هذا يمنع فتح أكثر من شاشة قفل واحدة فوق بعضها — وهو خطأ شائع يسبب مشاكل بصرية كبيرة.

before_sleep_cmd

ينفَّذ قُبيل دخول الجهاز في وضع السكون. loginctl lock-session يقفل الجلسة حتى يرى المستخدم شاشة القفل حين يعود ويفتح الغطاء.

after_sleep_cmd

ينفَّذ حين يستيقظ الجهاز من السكون. الأمر hyprctl dispatch dpms on يعيد تشغيل الشاشة. بعض الإعدادات تحتاج هذا الأمر وبعضها لا — جرّب وشوف.

شرح قسم listener

كل listener يحتوي على ثلاثة عناصر:

  • timeout — المدة بالثواني قبل تنفيذ الأمر
  • on-timeout — الأمر الذي يُنفَّذ عند انتهاء المهلة
  • on-resume — الأمر الذي يُنفَّذ حين يعود المستخدم (اختياري)
💡 يمكنك إضافة عدد غير محدود من الـ listeners. وبما أن الأوامر نصية عادية، يمكنك ربط سكريبتات كاملة بها — مثل تبديل وضع GPU أو تغيير الثيم عند اكتشاف انخفاض البطارية.

الـ Config الكامل الموصى به

هذا الإعداد يمر بأربع مراحل متتالية: تعتيم الشاشة، ثم إطفاؤها، ثم القفل، ثم التعليق بعد نصف ساعة. 

general {
    lock_cmd = pidof hyprlock || hyprlock
    before_sleep_cmd = loginctl lock-session
    after_sleep_cmd = hyprctl dispatch dpms on
}

# المرحلة 1: تعتيم الشاشة بعد 2.5 دقيقة
listener {
    timeout = 150
    on-timeout = brightnessctl -s set 10
    on-resume = brightnessctl -r
}

# المرحلة 2: قفل الجلسة بعد 5 دقائق
listener {
    timeout = 300
    on-timeout = loginctl lock-session
}

# المرحلة 3: إطفاء الشاشة بعد 5.5 دقائق
listener {
    timeout = 330
    on-timeout = hyprctl dispatch dpms off
    on-resume = hyprctl dispatch dpms on && brightnessctl -r
}

# المرحلة 4: تعليق النظام بعد 30 دقيقة
listener {
    timeout = 1800
    on-timeout = systemctl suspend
}

ترتيب المراحل مهم

لاحظ أن إطفاء الشاشة (330 ثانية) يأتي بعد القفل (300 ثانية). يمكنك عكس الترتيب حسب تفضيلك — أو حتى دمجهما في نفس الـ listener باستخدام &&: 

listener {
    timeout = 300
    on-timeout = loginctl lock-session && hyprctl dispatch dpms off
    on-resume = hyprctl dispatch dpms on
}

اختبار الإعداد بدون انتظار

لا تنتظر 5 دقائق لتختبر إعداداتك — غيّر الـ timeout مؤقتاً إلى 10 ثوانٍ واختبر: 

# تشغيل Hypridle يدوياً في الخلفية للاختبار
hypridle & disown

وإذا أردت التأكد من أنه يعمل فعلاً: 

pgrep hypridle

إذا أعاد رقماً (PID)، فهو يعمل. بعد الانتهاء من الاختبار، أعد الـ timeouts لقيمها الأصلية وأعد تشغيل Hyprland.

مشاكل شائعة وحلولها

Hypridle لا يعمل أبداً

الأسباب المحتملة:

  • نسيت إضافة exec-once = hypridle في إعدادات Hyprland
  • ملف hypridle.conf غير موجود أو فيه خطأ في الصياغة
  • تعارض بين نسخة hypridle ونسخة hyprland (git vs stable)

تحقق بتشغيله يدوياً من الطرفية لتشاهد أي خطأ: 

hypridle

الشاشة تُطفأ لكن لا يحدث قفل (أو العكس)

راجع ترتيب الـ listeners وقيم الـ timeout. إذا كان timeout القفل أكبر من timeout إطفاء الشاشة، ستُطفأ الشاشة أولاً ثم يحدث القفل — وهذا قد لا يكون ما تريده. قرر أي السلوكين تريد أن يأتي أولاً وعدّل الأرقام بناءً على ذلك.
