ملفات إعداد Hyprland أصبحت بلغة Lua 🌙

منذ الإصدار 0.55، ملف الإعداد أصبح Lua حقيقية — ليس DSL مخصص، بل لغة برمجة كاملة تغير كل شيء في طريقة التعامل مع المكونات.

١ مايو ٢٠٢٦

لماذا كان التغيير ضرورياً؟

لسنوات، كان كل مستخدمي Hyprland يعملون بنفس الطريقة: يفتحون hyprland.conf ويكتبون key-value pairs بصيغة Hyperlang. بسيطة، مألوفة، وتؤدي الغرض.

لكن مع نمو Hyprland، أصبحت المشكلة واضحة. قواعد النوافذ تحولت من سطر بسيط إلى وحوش كهذا: 

# window rule قديمة — كابوس حقيقي
windowrulev2 = immediate,bordersize 4,class:^(Among Us)$,title:^(Among Us)$

Basti (مطور Hyprland) نفسه وصف صيغة Hyperlang بأنها "غير موثوقة، غير مقروءة، محدودة، ومفككة". المشكلة الأساسية لم تكن القبح — كانت أن صيغة key-value لا تستطيع حمل منطق:

  • لا يمكن وضع if داخل keybind
  • لا يمكن عمل loop على قائمة workspaces
  • لا يمكن تعريف دالة تُنادى عند فتح نافذة
  • كل سلوك شرطي يحتاج سكريبت bash خارجي

Lua تحل كل هذا. منذ الإصدار 0.55 — ملف الإعداد أصبح Lua.

لماذا Lua تحديداً؟

Lua ليست اختياراً عشوائياً:

  • خفيفة وسريعة جداً — مصممة للتضمين داخل تطبيقات
  • مُستخدمة في Neovim وAwesomeWM وكثير من الأدوات
  • متغيرات، دوال، حلقات، شروط، جداول، ومكتبة قياسية كاملة
  • مألوفة لكثير من مستخدمي Linux المتقدمين

الانتقال إلى الصيغة الجديدة

موقع ملف الإعداد الجديد

~/.config/hypr/hyprland.lua

عند تشغيل Hyprland، إذا وجد هذا الملف يحمّله بدلاً من hyprland.conf. إذا لم يكن موجوداً، يرجع للصيغة القديمة — لا شيء ينكسر إذا لم تهاجر بعد.

الـ Keybinds — قبل وبعد

# قبل — Hyperlang
bind = SUPER, Return, exec, kitty
-- بعد — Lua
hl.bind("SUPER", "Return", hl.dsp.exec_cmd, "kitty")

بسيطة، لكن الفرق الحقيقي يظهر عند إضافة منطق:

-- keybind بمنطق شرطي — مستحيل في الصيغة القديمة
hl.bind("SUPER", "X", function()
  local w = hl.get_active_window()
  if w ~= nil and w.title == "htop" then
    hl.dsp.window.float({ action = "set" })
  else
    hl.dsp.window.float({ action = "toggle" })
  end
end)

قواعد النوافذ — قبل وبعد

# قبل — سطر طويل لا يُقرأ
windowrulev2 = float,center,size 200 300,class:^(font-switcher)$
-- بعد — جدول Lua واضح ومقروء
hl.window_rule({
  match = { class = "font-switcher" },
  float = true,
  center = true,
  size = { w = 200, h = 300 }
})

في الصيغة القديمة، conditions وeffects كانت محشورة في سطر واحد بفواصل وregex. في Lua، كل شيء في جدول بمفاتيح مسماة — يمكن قراءته وفهمه بلمحة واحدة. ومحرر الكود يساعدك: syntax highlighting، LSP، وkشف الأخطاء — مجاناً.

الإمكانيات الجديدة

١. Event Callbacks — ردود فعل على أحداث المكون

الآن يمكنك تسجيل دوال تُنادى عند وقوع أحداث في compositor: 

-- إشعار عند كل تغيير focus
hl.on("window.active", function(w)
  hl.notification.create({
    text = "focused: " .. w.title,
    timeout = 3000,
    icon = "ok"
  })
end)

فكّر في الاحتمالات:

  • تطبيق إعدادات مختلفة تلقائياً عند focus نافذة معينة
  • تغيير الثيم حسب الوقت
  • تسجيل التطبيقات الأكثر استخداماً

٢. Timers — جدولة مهام بدون cron

-- ينفذ كل دقيقة بدون cron job أو daemon خارجي
local my_timer = hl.timer(function()
  -- افعل شيئاً دورياً هنا
end, {
  timeout = 60000,  -- milliseconds
  type = "repeat"
})

يمكنك استخدام هذا لتدوير الخلفيات، تغيير الثيم حسب الوقت، فحص مستوى البطارية وتعديل الإعدادات — كل هذا من ملف الإعداد مباشرة.

٣. Configs مجزأة ومنظمة

بدلاً من صيغة source القديمة، تستخدم require من Lua: 

-- hyprland.lua الرئيسي
require("modules.keybinds")
require("modules.window-rules")
require("modules.animations")
require("modules.theme")
-- هيكل المجلدات
~/.config/hypr/
├── hyprland.lua
└── modules/
    ├── keybinds.lua
    ├── window-rules.lua
    ├── animations.lua
    └── theme.lua

الفائدة: بدلاً من ملف 450 سطر تبحث فيه عن إعداد واحد، كل جزء في ملفه المنطقي. عند تعديل keybind تفتح keybinds.lua مباشرة.

كيف تهاجر من الصيغة القديمة؟

  1. احتفظ بـ hyprland.conf كما هو — لن ينكسر شيء
  2. أنشئ hyprland.lua فارغاً وابدأ بنقل أقسام واحداً بواحد
  3. ابدأ بالأقسام البسيطة (الإعدادات العامة، الألوان) قبل الانتقال للـ keybinds والقواعد
  4. اختبر بعد كل مرحلة: hyprctl reload
⚠️ ملاحظة: Hyprland لا يحمّل الملفين معاً — إذا وجد hyprland.lua يتجاهل hyprland.conf. احذف أو أعد تسمية hyprland.lua للرجوع للصيغة القديمة.

الخلاصة

هذا ليس مجرد تغيير في الصيغة — هو تحويل Hyprland من "ملف إعداد" إلى "برنامج يصف بيئتك". ملف الإعداد أصبح قادراً على التفاعل مع المكون، التفكير، واتخاذ قرارات.

AwesomeWM كان دائماً المثال على ما يمكن فعله بـ Lua في window manager. الآن Hyprland يمشي في نفس الاتجاه — مع الأداء والجماليات التي نحبها.

🎯 إذا كنت تستخدم Hyprland 0.55+، ابدأ بالتهجرة التدريجية. الإمكانيات الجديدة تستحق.
