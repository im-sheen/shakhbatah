️ Submaps في Hyprland — أوضاع المفاتيح المتقدمة

أنشئ أوضاع مفاتيح مخصصة مثل Vim تماماً — لتمرير VM، تغيير أحجام النوافذ، التحكم في الميديا، وأكثر.

٧ سبتمبر ٢٠٢٥

المشكلة — VM يسرق مفاتيحك

إذا جرّبت تشغيل window manager داخل VM — سواء i3 أو Sway أو DWM أو حتى Hyprland نفسه — ستواجه مشكلة مزعجة: الـ host يعترض مفاتيحك قبل أن تصل للـ VM. تضغط Super+1 لتبديل workspace في الـ guest، فيتحرك الـ host بدلاً منه. أنت عالق تستخدم الفأرة مثل... شخص عادي.

الحل هو Submaps — ميزة في Hyprland تتيح لك إنشاء أوضاع منفصلة لمفاتيحك، مثل أوضاع Vim تماماً.

ما هو الـ Submap؟

فكّر في الأمر هكذا: عادةً أنت في "الوضع العادي" حيث Super+D يفتح المشغّل، Super+1 يبدّل الـ workspace، وهكذا. الـ Submap يسمح لك بالدخول لـ "وضع آخر" حيث نفس المفاتيح تفعل أشياء مختلفة تماماً، أو لا تفعل شيئاً أصلاً.

الفكرة هي نفس أوضاع Vim: Normal Mode، Insert Mode، Visual Mode — كل وضع له قواعده.

البنية الأساسية

# الدخول للـ submap
bind = $mainMod, R, submap, resize

# --- بداية الـ submap ---
submap = resize

  bind = , right, resizeactive, 30 0
  bind = , left,  resizeactive, -30 0
  bind = , up,    resizeactive, 0 -30
  bind = , down,  resizeactive, 0 30

  # الخروج بـ Escape
  bind = , escape, submap, reset

  # مهم جداً: Catch-All لتجنب الانحشار
  bind = , catchall, submap, reset

# --- نهاية الـ submap ---
submap = reset
⚠️ تحذير مهم: إذا دخلت submap ولم يكن عندك مفتاح للخروج، ستنحشر فيه. الحل الوحيد سيكون التبديل لـ TTY آخر وقتل Hyprland. دائماً أضف catchall أو مفتاح خروج.

حالة الاستخدام الأولى — تمرير VM (Passthrough)

هذه الحالة التي بدأنا بها. ندخل وضعاً يمرّر كل المفاتيح مباشرة للتطبيق بدون أن يعترضها Hyprland: 

# في binds.conf

# الدخول لوضع VM passthrough
bind = $mainMod, V, submap, passthrough

submap = passthrough

  # تمرير كل المفاتيح للتطبيق
  bind = , catchall, pass, foreground

  # الخروج من الوضع بـ Super+V مرة أخرى
  bind = $mainMod, V, submap, reset

submap = reset

الآن حين تضغط Super+V يدخل Hyprland وضع "تمرير كامل". كل ما تضغطه يذهب للـ VM. تضغط Super+V مرة ثانية للخروج والعودة للوضع العادي.

يمكنك إضافة إشعار لتعرف في أي وضع أنت:

bind = $mainMod, V, exec, notify-send "VM Mode" "كل المفاتيح تذهب للـ VM"
bind = $mainMod, V, submap, passthrough

submap = passthrough
  bind = , catchall, pass, foreground
  bind = $mainMod, V, exec, notify-send "Host Mode" "Hyprland يتحكم بالمفاتيح"
  bind = $mainMod, V, submap, reset
submap = reset

حالة الاستخدام الثانية — تغيير حجم النوافذ

بدلاً من الضغط المستمر على مفتاح modifier لتغيير الحجم، ادخل وضع resize وتحرر:

bind = $mainMod, R, submap, resize

submap = resize

  # e = repeatable — يتكرر عند الضغط المستمر
  binde = , right, resizeactive, 30 0
  binde = , left,  resizeactive, -30 0
  binde = , up,    resizeactive, 0 -30
  binde = , down,  resizeactive, 0 30

  # HJKL لمن يحب Vim
  binde = , l, resizeactive, 30 0
  binde = , h, resizeactive, -30 0
  binde = , k, resizeactive, 0 -30
  binde = , j, resizeactive, 0 30

  bind = , escape, submap, reset
  bind = , catchall, submap, reset

submap = reset
💡 binde بدلاً من bind تعني "repeatable" — يتكرر التأثير طالما المفتاح مضغوط. مهمة جداً لتغيير الحجم.

حالة الاستخدام الثالثة — إدارة النوافذ

وضع مخصص لإجراءات النوافذ الشائعة:

bind = $mainMod, W, submap, window-mgmt

submap = window-mgmt

  # تبديل fullscreen
  bind = , F, fullscreen, 0

  # تبديل floating
  bind = SHIFT, F, togglefloating

  # تحريك النافذة بالكيبورد
  bind = , right, movewindow, r
  bind = , left,  movewindow, l
  bind = , up,    movewindow, u
  bind = , down,  movewindow, d

  # إغلاق النافذة
  bind = , Q, killactive

  bind = , escape, submap, reset
  bind = , catchall, submap, reset

submap = reset

حالة الاستخدام الرابعة — التحكم في الميديا

وضع للتحكم في الموسيقى دون الحاجة لمفاتيح وظيفية أو تركيبات معقدة:

bind = $mainMod, M, submap, media

submap = media

  # التحكم الأساسي
  bind = , P,      exec, playerctl play-pause
  bind = , right,  exec, playerctl next
  bind = , left,   exec, playerctl previous

  # أو بـ I, O, P (أسهل للوصول)
  bind = , I, exec, playerctl previous
  bind = , O, exec, playerctl play-pause
  bind = , bracketright, exec, playerctl next

  # الصوت
  binde = , up,   exec, wpctl set-volume @DEFAULT_AUDIO_SINK@ 5%+
  binde = , down, exec, wpctl set-volume @DEFAULT_AUDIO_SINK@ 5%-

  bind = , escape, submap, reset
  bind = , catchall, submap, reset

submap = reset

يمكنك إظهار اسم الـ submap النشط في شريط Waybar حتى تعرف دائماً في أي وضع أنت. في إعدادات Waybar أضف:

الآن حين تدخل أي submap سيظهر اسمه في الشريط — مثلاً resize أو media.

إذا انحشرت في Submap

إذا حدث ونسيت مفتاح الخروج أو بدون catchall، هذا الأمر ينقذك:

hyprctl dispatch submap reset

شغّله من طرفية أخرى أو من TTY آخر (Ctrl+Alt+F2).

ملف Submaps كامل جاهز للاستخدام

# ============================

# VM Passthrough
bind = $mainMod, V, exec, notify-send "🖥️ VM Mode" "المفاتيح تذهب للـ VM"

submap = passthrough
  bind = , catchall, pass, foreground
  bind = $mainMod, V, exec, notify-send "💻 Host Mode" "Hyprland يتحكم"

submap = reset

الخلاصة

Submaps ميزة قوية ومغمورة. معظم مستخدمي Hyprland لا يعرفون عنها، وهي تحل مشاكل حقيقية — أبرزها مشكلة VM passthrough التي تجعل تشغيل window manager داخل VM أمراً ممكناً فعلاً.

لكنها أيضاً طريقة لتنظيم مفاتيحك بشكل أكثر منطقية — بدلاً من تركيبات معقدة مثل Super+Shift+Ctrl+R، يمكنك الدخول لوضع resize والضغط على السهم فقط.

🎯 القاعدة الذهبية: دائماً أضف bind = , catchall, submap, reset في كل submap تنشئه. ستشكر نفسك لاحقاً.
