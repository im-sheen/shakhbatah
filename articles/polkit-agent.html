إصلاح نوافذ كلمة المرور في Hyprland

لماذا لا تعمل نوافذ كلمة المرور وكيف تضبط hyprpolkit أو gnome-polkit لتعمل بشكل صحيح.

١٦ مايو ٢٠٢٦

المشكلة

حين تحاول تشغيل أمر يحتاج صلاحيات root (مثل pkexec)، لا تظهر نافذة كلمة المرور. السبب: Hyprland لا يأتي مع polkit authentication agent مثبتاً — يجب تثبيته يدوياً.

ما هو Polkit؟

Polkit (Policy Kit) هو الإطار المسؤول عن إظهار نافذة كلمة المرور حين يطلب تطبيق ما صلاحيات مرتفعة. بدون agent، هذه النافذة لن تظهر أبداً.

الخيار الأول: gnome-polkit (موصى به لمستخدمي GTK)

sudo pacman -S polkit-gnome

أضفه في autostart:

exec-once = /usr/lib/polkit-gnome/polkit-gnome-authentication-agent-1

النافذة الناتجة تتبع ثيم GTK الخاص بك تلقائياً.

الخيار الثاني: hyprpolkit (لمستخدمي Qt)

yay -S hyprpolkitagent

تشغيله عبر systemd:

# بدء الخدمة
systemctl --user start hyprpolkitagent.service

# إضافة لـ autostart
exec-once = systemctl --user start hyprpolkitagent.service

مقارنة بين الخيارين

gnome-polkithyprpolkit
التخصيصيتبع GTK themeيتبع Qt/QML
الاستخدام الأنسبمعظم إعدادات Hyprlandمن يستخدم Qt apps كثيراً
السهولةأبسطيحتاج systemd
💡 إن كنت تستخدم GTK apps في الغالب (Thunar, Gedit, إلخ)، اختر gnome-polkit — أبسط وأجمل.

اختبار الإعداد

# اختبر أن الـ agent يعمل
pkexec echo "يعمل!"

يجب أن تظهر نافذة كلمة المرور — إن ظهرت فالإعداد ناجح.
