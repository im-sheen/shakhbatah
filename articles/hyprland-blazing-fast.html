اجعل Hyprland أسرع ما يكون

كيف تضيف قائمة Power Profiles لـ Hyprland وتربطها بـ Waybar، مع تقنيات أتمتة إضافية لتسريع سير العمل اليومي.

١٧ يناير ٢٠٢٦

المشكلة — لا توجد قائمة Power Profiles خارج بيئات سطح المكتب

في GNOME توجد قائمة Power Profiles مدمجة في مركز التحكم — تختار بين Performance وBalanced وPower Saver بنقرة واحدة. في Hyprland لا يوجد شيء كهذا افتراضياً.

لكن الحل موجود ويعمل بنفس الكفاءة تماماً بل أفضل، لأنك تتحكم به من Waybar ومن لوحة المفاتيح مباشرةً.

الأداة: power-profiles-daemon

power-profiles-daemon هي الخدمة المسؤولة عن إدارة بروفايلات الطاقة في لينكس. تعمل في الخلفية وتوفر ثلاثة بروفايلات:

  • performance — أقصى أداء، استهلاك أكبر للبطارية
  • balanced — التوازن الافتراضي بين الأداء والبطارية
  • power-saver — توفير البطارية، أداء أقل

التثبيت والتفعيل

# على Arch Linux
sudo pacman -S power-profiles-daemon

# تفعيل الخدمة
sudo systemctl enable --now power-profiles-daemon
💡 بعد التثبيت ستجد الأمر powerprofilesctl متاحاً في الترمنل.

أوامر powerprofilesctl

# عرض البروفايلات المتاحة (البروفايل الحالي عليه *)
powerprofilesctl list

# عرض البروفايل الحالي فقط
powerprofilesctl get

# تغيير البروفايل
powerprofilesctl set performance
powerprofilesctl set balanced
powerprofilesctl set power-saver

مثال على الناتج:

* performance:
    Driver:     amd_pstate
    Degraded:   no

  balanced:
    Driver:     amd_pstate

  power-saver:
    Driver:     amd_pstate

إضافة Power Profile Module إلى Waybar

Waybar يحتوي على module جاهز لـ power-profiles-daemon. أضفه إلى إعدادات الـ bar:

config.jsonc

{
  "modules-right": [
    "power-profiles-daemon",
    "battery",
    "clock"
  ],

  "power-profiles-daemon": {
    "format": "{icon}",
    "tooltip-format": "Power profile: {profile}\nDriver: {driver}",
    "tooltip": true,
    "format-icons": {
      "default":      "⚡",
      "performance":  "🚀",
      "balanced":     "⚖️",
      "power-saver":  "🌿"
    }
  }
}

الآن تظهر الأيقونة في Waybar وتتغير تلقائياً مع كل تغيير في البروفايل — سواء من الترمنل أو من الـ keybind.

ربط البروفايلات بـ Keybind — "Blitz Mode"

الفكرة: اضغط Super + F1 للتبديل بين وضع الأداء الكامل ووضع توفير البطارية. مفيد جداً على اللاب توب.

أنشئ سكريبت التبديل في ~/.local/bin/toggle-power-profile:

#!/usr/bin/env bash

CURRENT=$(powerprofilesctl get)

if [ "$CURRENT" = "power-saver" ]; then
  powerprofilesctl set performance
  notify-send "⚡ Performance Mode" "تم تفعيل وضع الأداء الكامل"
else
  powerprofilesctl set power-saver
  notify-send "🌿 Power Saver" "تم تفعيل وضع توفير البطارية"
fi

chmod +x ~/.local/bin/toggle-power-profile

ثم في ملف الـ keybinds:

# ~/.config/hypr/keybinds.conf

bind = SUPER, F1, exec, ~/.local/bin/toggle-power-profile
🎯 يمكنك توسيع هذا السكريبت ليطفئ الـ animations والـ blur عند دخول Power Saver Mode عبر hyprctl keyword — مما يزيد العمر الافتراضي للبطارية بشكل ملحوظ.

إطفاء الأنيميشن والـ Blur في Power Saver Mode

hyprctl keyword يتيح تغيير إعدادات Hyprland مؤقتاً دون إعادة تحميل الكونفيج. أضف هذا للسكريبت: 

#!/usr/bin/env bash

CURRENT=$(powerprofilesctl get)

if [ "$CURRENT" = "power-saver" ]; then
  # تفعيل وضع الأداء
  powerprofilesctl set performance
  hyprctl keyword animations:enabled true
  hyprctl keyword decoration:blur:enabled true
  notify-send "⚡ Performance Mode" "الأنيميشن والبلور مفعّلان"
else
  # تفعيل توفير البطارية
  powerprofilesctl set power-saver
  hyprctl keyword animations:enabled false
  hyprctl keyword decoration:blur:enabled false
  notify-send "🌿 Power Saver" "الأنيميشن والبلور مُطفآن"
fi

تسريع سير العمل — Aliases في ZSH

جانباً من موضوع Power Profiles، هناك طريقة بسيطة جداً لجعل الترمنل أسرع في الاستخدام اليومي: الـ Aliases.

الـ alias هو اختصار تعيّنه لأمر طويل. فكر فيه كـ shortcut في ويندوز لكن داخل الترمنل. 

# أضف هذا في ~/.zshrc

# تثبيت الحزم بحرف واحد
alias i="yay -S"

# حذف الحزم مع dependencies
alias r="yay -Rns"

# تحديث النظام كاملاً
alias up="yay -Syu"

# التنقل السريع
alias ..="cd .."
alias ...="cd ../.."

# git مختصر
alias gs="git status"
alias ga="git add ."
alias gc="git commit -m"
alias gp="git push"

# لتفعيلها فوراً
source ~/.zshrc

مثال على الاستخدام:

# بدلاً من: yay -S neovim
i neovim

# بدلاً من: yay -Rns cmatrix
r cmatrix

# بدلاً من: yay -Syu
up

تقنية sudo nmcli لتجنب كلمة المرور المتكررة

عند تشغيل تحديث النظام وتريد تركه يعمل وحده بدون أن تظل منتظراً لإدخال كلمة المرور، هناك حيلة بسيطة: شغّل أمر sudo مرة واحدة أولاً لتفتح جلسة sudo، ثم شغّل التحديث. 

# افتح جلسة sudo أولاً (يطلب كلمة المرور مرة واحدة)
sudo -v

# الآن شغّل التحديث بدون توقف
yay -Syu --noconfirm

أو استخدم --noconfirm مباشرة مع --needed في سكريبتات الـ dotfiles لتثبيت الحزم دفعة واحدة دون أي تدخل: 

sudo pacman -S --needed --noconfirm \
  hyprland \
  waybar \
  kitty \
  rofi-wayland \
  dunst
💡 --needed لا تثبّت الحزمة إذا كانت موجودة بالفعل — مفيد جداً عند إعادة تشغيل سكريبت الـ dotfiles على نظام مثبّت مسبقاً.

خلاصة

لجعل Hyprland أسرع وأذكى في إدارة الأداء والعمل اليومي:

  1. ثبّت power-profiles-daemon وفعّل الخدمة
  2. أضف module في Waybar يعرض البروفايل الحالي
  3. أنشئ سكريبت toggle-power-profile للتبديل بضغطة مفتاح
  4. وسّع السكريبت ليطفئ الـ animations والـ blur في Power Saver تلقائياً
  5. استخدم الـ aliases في ZSH للأوامر الطويلة المتكررة
  6. استخدم --needed --noconfirm في سكريبتات الـ dotfiles
