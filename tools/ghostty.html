Ghostty 👻 — هل هذا مستقبل طرفيات Hyprland؟

مراجعة كاملة من منظور مستخدم Hyprland — لا مراجعة عامة، بل إجابة على الأسئلة التي تهمنا فعلاً.

٢٣ أبريل ٢٠٢٦

ما هو Ghostty؟

Ghostty هو محاكي طرفية أُطلق في ديسمبر 2024، وصل إلى 50,000 نجمة على GitHub في أقل من سنة، وأُضيف إلى مستودعات Ubuntu الافتراضية. بناه Mitchell Hashimoto — الشخص الذي أنشأ Terraform وVagrant — وكُتب بالكامل بلغة Zig مع تسريع GPU للرسم.

لكن السؤال الذي يهمنا نحن مستخدمي Hyprland: هل يستحق التبديل إليه، أم أنه مجرد ضجة من مجتمع macOS؟

🎯 هذه ليست مراجعة عامة — هي إجابة على الأسئلة التي تهم مستخدمي Hyprland تحديداً: كيف يعمل على Wayland؟ ما مدى قابليته للتخصيص؟ وكيف يقارن بـ Kitty وFoot؟

البنية التقنية

Ghostty مبني بشكل مختلف عن معظم الطرفيات:

  • اللغة: Zig — لغة أنظمة تعطيك أداء C بدون مشاكله
  • على Linux: تطبيق GTK4 + libadwaita حقيقي مع OpenGL rendering
  • على macOS: تطبيق SwiftUI حقيقي مع Metal rendering
  • Multi-threaded: خيط منفصل للقراءة والكتابة والرسم لكل instance
  • الترخيص: MIT مفتوح المصدر، يُدار كـ nonprofit بدون شركة خلفه

الفارق المهم: Ghostty ليس wrapper متعدد المنصات — هو مبني مع Linux graphics stack في الاعتبار من البداية.

دعم Wayland

Ghostty يعمل على Wayland بشكل أصلي عبر GTK4. لا يوجد XWayland fallback. هذا يعني:

  • تكبير/تصغير صحيح على شاشات HiDPI
  • تعامل سليم مع input Hyprland
  • تكامل كامل مع قواعد النوافذ window rules
  • يتصرف كـ "مواطن Wayland محترم" — بدون مشاكل التوافق الشائعة

بروتوكول Kitty للصور

Ghostty يدعم Kitty Graphics Protocol بالكامل، يعني عرض الصور مباشرة داخل الطرفية يعمل. إذا كنت تستخدم أدوات مثل yazi أو lf مع معاينة الصور، أو تعرض مخططات على الطرفية — كل هذا يعمل بشكل مثالي. 

# اختبار بروتوكول الصور
kitten icat ~/Pictures/wallpaper.png

جودة عرض الخطوط

عرض الخطوط في Ghostty ممتاز ويشبه جداً ما تراه في Kitty:

  • دعم كامل للـ ligatures
  • Nerd Fonts تعمل out of the box
  • Unicode و grapheme clustering صحيحَان — حتى الـ emoji المعقدة تظهر كحرف واحد

إذا قارنت بـ Foot: Foot أكثر حدة وصراحة، بينما Ghostty وKitty لهما نعومة مشابهة لما تراه على macOS. الفرق خفي جداً إلا لمن يلاحظ التفاصيل.

الإعداد والتخصيص

ملف الإعداد في ~/.config/ghostty/config — بدون JSON أو YAML أو TOML، فقط key = value سطر بسطر. 

# مثال على إعداد Ghostty
font-family = JetBrainsMono Nerd Font
font-size = 13
theme = catppuccin-mocha
background-opacity = 0.95
window-padding-x = 12
window-padding-y = 8

لتطبيق التغييرات فوراً بدون إعادة تشغيل: Ctrl+Shift+,

الثيمات المدمجة

Ghostty يأتي مع مئات الثيمات المدمجة. تجدها في: 

ls /usr/share/ghostty/themes/

لتفعيل ثيم:

theme = catppuccin-mocha

يدعم أيضاً ثيمات منفصلة للـ light/dark mode، وهو مفيد جداً مع نظام تبديل الثيمات الديناميكي. كل ما تحتاجه هو template يكتب في ملف الإعداد عند تغيير الثيم.

هل يستحق التبديل؟

نعم، إذا:

  • تريد طرفية GTK4 تندمج بشكل طبيعي مع ثيمات libadwaita
  • تريد ثيمات مدمجة كثيرة بدون إعداد إضافي
  • تستخدم yazi أو أدوات تحتاج Kitty Graphics Protocol
  • تريد إعاداً ديناميكياً مع light/dark mode

ابقَ مع Kitty، إذا:

  • عندك إعداد Kitty متقدم تعبت في بنائه
  • تستخدم Kitty kittens (أدوات Kitty المدمجة)

ابقَ مع Foot، إذا:

  • تريد أقل استهلاك موارد ممكن
  • لا تحتاج Kitty Graphics Protocol
💡 الخلاصة: Ghostty ليس مجرد hype. هو طرفية حقيقية مبنية بجدية لـ Linux، وتدعم Wayland بشكل أصلي. إذا كنت تبحث عن بديل لـ Kitty أو تجربة جديدة — يستحق التجربة.
