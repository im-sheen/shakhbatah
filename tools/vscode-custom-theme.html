صنع ثيم VS Code مخصص من الصفر 🎨

عندك ثيم جميل على لينكس لكن ما له نسخة VS Code؟ هذه المقالة تشرح كيف تصنع امتداد ثيم كامل بنفسك — بمساعدة الذكاء الاصطناعي.

٢٠ يناير ٢٠٢٤

المشكلة

الكثير من ثيمات لينكس الجميلة — خاصة المشهورة على r/unixporn — ليس لها نسخة على VS Code أو VS Codium. يعني إعداد سطح المكتب بالكامل يكون متناسقاً، لكن بمجرد ما تفتح المحرر ينكسر التناسق.

الحل: تصنع امتداد ثيم خاص بك، وهو أسهل مما يبدو.

💡 VS Code أم VS Codium؟ كلاهما متطابق من ناحية الامتدادات والثيمات. الفرق الوحيد أن VS Codium محذوف منه كل كود التتبع والـ telemetry الخاص بـ Microsoft. كل ما في هذه المقالة يعمل على الاثنين.

المتطلبات

  • Node.js و npm مثبتَين على الجهاز
  • ألوان الثيم الذي تريد تحويله (سنشرح كيف تحصل عليها)
  • أي نموذج ذكاء اصطناعي (ChatGPT، Claude، Gemini، إلخ)

الخطوة الأولى: اختر ثيمك واستخرج ألوانه

القاعدة الأولى في اختيار الثيم: كلما زاد عدد الألوان كان أفضل. لأنك ستُصمم محرر نصوص يحتاج ألواناً مختلفة للـ syntax highlighting، وكلما كان لديك تشكيلة ألوان أوسع كان الناتج أجمل.

الخيار المثالي هو ثيمات Base46 — وهي ثيمات تحتوي على 46 لوناً مصنفاً. المصدر الأفضل لها هو NvChad، وهو إعداد جاهز لـ Neovim. حتى لو لا تستخدم Neovim، ثيماته مثالية لهذا الغرض.

كيف تحصل على ملف الألوان من NvChad

إذا كان NvChad مثبتاً، ستجد ملفات الألوان هنا:

~/.local/share/nvim/lazy/base46/lua/base46/themes/

داخل هذا المجلد ملف Lua لكل ثيم. افتح ملف الثيم الذي تريده وستجد جميع الألوان مُعرَّفة بشكل واضح — 30 لوناً للـ syntax و16 للطرفية.

🔍 إذا لم يكن NvChad مثبتاً، يمكنك تصفح ملفاته مباشرة على GitHub: github.com/NvChad/base46 ثم lua/base46/themes/

الخطوة الثانية: إنشاء هيكل الامتداد

VS Code يوفر أداة تولّد هيكل الامتداد تلقائياً. ثبّتها أولاً: 

sudo npm install -g yo generator-code

الـ -g مهمة — تعني التثبيت بشكل عام (global) حتى تعمل الأداة من أي مجلد. بدونها ستُثبَّت داخل مجلد node_modules المحلي ولن تتمكن من استخدامها.

بعدها أنشئ مجلداً للمشروع وادخل إليه:

mkdir my-theme && cd my-theme

ثم شغّل المولّد:

yo code

سيسألك عدة أسئلة، هذه إجاباتها:

السؤالالجواب
What do you want to do?اختر: New Color Theme
Import or convert existing?اختر: No, start fresh
Extension nameاسم ثيمك، مثلاً: rayan-dark
Identifierنفس الاسم بالـ lowercase
Descriptionوصف مختصر
Theme name (shown to user)الاسم كما سيظهر في VS Code
Base themeDark
Initialize git repo?حسب رأيك — No يكفي للاستخدام الشخصي

بعد الانتهاء سيُنشئ المولّد مجلداً بهذا الهيكل:

my-theme/
├── package.json
└── themes/
    └── my-theme-color-theme.json   ← هذا هو الملف المهم

الخطوة الثالثة: استبدال الألوان بالذكاء الاصطناعي

ملف themes/my-theme-color-theme.json يحتوي على نحو 600 سطر من إعدادات الألوان الافتراضية لـ VS Code. مهمتك هي استبدال هذه الألوان بألوان ثيمك — وهذا بالضبط ما سيفعله الذكاء الاصطناعي.

المطلوب: استبدل ألوان ملف JSON بالألوان الموجودة في ملف Lua.
قم أيضاً بتثيم:
- واجهة المحرر بالكامل (Editor UI)
- لوحة الأوامر (Command Palette / Quick Input)
- ألوان أقواس البرمجة (Bracket Pair Colorization)

أعطني ملف JSON الكامل بعد الاستبدال.
⚠️ ملاحظة مهمة: إذا استخدمت نموذجاً مجانياً أو في وضع incognito، قد يُقصّر الإجابة ولا يعطيك الملف كاملاً. استخدم نموذجاً مدفوعاً أو على الأقل حساباً مسجلاً للحصول على الملف كاملاً دون انقطاع.

الأقسام التي يجب التأكد من تثيمها

بعد الحصول على الـ JSON من الذكاء الاصطناعي، تأكد أن هذه الأقسام مثيّمة:

  • Syntax highlighting — ألوان الكلمات المفتاحية والمتغيرات والتعليقات في الكود. هذا ما يثيّمه الذكاء الاصطناعي أولاً تلقائياً.
  • Editor UI — خلفية المحرر، شريط الجانب، شريط الحالة، شريط النشاط. إذا تُركت بدون تثيم تبدو غريبة مع بقية الإعدادات.
  • Quick Input / Command Palette — لوحة الأوامر التي تفتح بـ Ctrl+Shift+P. إذا لم تُثيّمها ستظهر بلون مختلف تماماً.
  • Bracket Pair Colors — ألوان الأقواس المتداخلة في الكود. الألوان الافتراضية قد تبدو غريبة مع ثيمك الجديد.

في ملف JSON، هذه الأقسام تبدو هكذا:

{
  "colors": {
    // Editor UI
    "editor.background": "#1a1b26",
    "editor.foreground": "#c0caf5",
    "editorLineNumber.foreground": "#3b4261",
    "activityBar.background": "#16161e",
    "statusBar.background": "#16161e",

    // Command Palette
    "quickInput.background": "#1a1b26",
    "quickInputList.focusBackground": "#2a2b3d",

    // Bracket colors
    "editorBracketHighlight.foreground1": "#c0caf5",
    "editorBracketHighlight.foreground2": "#bb9af7",
    "editorBracketHighlight.foreground3": "#7dcfff",
    "editorBracketHighlight.foreground4": "#9ece6a",
    "editorBracketHighlight.foreground5": "#ff9e64"
  },
  "tokenColors": [
    // Syntax highlighting هنا
  ]
}

الخطوة الرابعة: اختبار الثيم

قبل تثبيت الامتداد، اختبره مباشرة:

  1. افتح مجلد المشروع في VS Code: code . أو codium .
  2. افتح لوحة الأوامر: Ctrl+Shift+P
  3. اكتب: Debug: Start Debugging واضغط Enter
  4. سيفتح نافذة VS Code جديدة بثيمك مُفعَّلاً

تصفّح الواجهة الجديدة وتحقق من كل شيء — افتح ملف كود، افتح لوحة الأوامر، تأكد أن الألوان متناسقة في كل مكان. إذا وجدت شيئاً يبدو غريباً، عُد للذكاء الاصطناعي واطلب تعديل تلك الألوان تحديداً.

الخطوة الخامسة: تعبئة الامتداد وتثبيته

بعد الانتهاء من التثيم، حوّل المشروع إلى ملف امتداد قابل للتثبيت:

تثبيت أداة التعبئة

sudo npm install -g @vscode/vsce

تعبئة الامتداد

cd my-theme
vsce package --allow-missing-repository

الـ --allow-missing-repository ضرورية إذا لم تُهيّئ git repository. بعد تشغيل الأمر ستجد ملفاً بالامتداد .vsix في نفس المجلد.

تثبيت الامتداد

# لـ VS Code
code --install-extension my-theme-1.0.0.vsix

# لـ VS Codium
codium --install-extension my-theme-1.0.0.vsix

بعد التثبيت أغلق VS Code وأعد فتحه، ثم:

  1. افتح لوحة الأوامر: Ctrl+Shift+P
  2. اكتب: Preferences: Color Theme
  3. ابحث عن اسم ثيمك في القائمة واختره

نصيحة: كيف تحدّث الثيم لاحقاً

إذا أردت تعديل لون معين أو إضافة دعم لصيغة برمجية جديدة:

  1. عدّل ملف themes/my-theme-color-theme.json مباشرة
  2. في نافذة الاختبار، اضغط Ctrl+R لإعادة تحميل الامتداد
  3. عندما تنتهي، شغّل vsce package مجدداً وثبّت الملف الجديد
🎯 الخلاصة: الفكرة كلها هي — خذ ملف JSON الافتراضي، خذ ألوان ثيمك، اطلب من الذكاء الاصطناعي يستبدل الألوان، وعبّئ الامتداد. العملية باكملها تاخذ أقل من 30 دقيقة.
